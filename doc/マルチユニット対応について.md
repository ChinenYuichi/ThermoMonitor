# マルチユニット対応について

## 前提

CPSN-SSI-4C は最大4ユニット設置可能。現在のプログラムは Unit ID = 1 のみを対象としている。

---

## Unit ID の実装

`PANEL_UNIT_ID` はすでに UI で変更可能なパラメータであり、Modbus リクエストにそのまま反映される。

```c
// ReadAndDisplayTemperatures / StartMonitoring 内
GetNumericCtrlAsDouble(panelHandle, PANEL_UNIT_ID, &dUnitId);
unitId = (int)dUnitId;
SSI4C_ReadAllChannels(unitId, 0, chData, g_currentDataFormat);
```

```c
// Modbus_ReadInputRegisters 内
reqBuf[6] = (uint8_t)unitId;   // MBAP ヘッダの Unit ID フィールド
```

---

## ケース別の対応状況

### ケース①：Unit ID を変更して特定ユニットの4チャンネルを表示する

**✅ 現状のままで対応可能。改修不要。**

UI の `Unit ID` 欄を変更して接続するだけで、対象ユニットの CH0〜CH3 が表示される。

| Unit ID 設定値 | 対象 |
|---|---|
| 1 | モジュール1（CH0〜CH3） |
| 2 | モジュール2（CH0〜CH3） |
| 3 | モジュール3（CH0〜CH3） |
| 4 | モジュール4（CH0〜CH3） |

**運用手順（切り替え）：**
```
Unit 1 を監視 → Unit ID = 1 → 接続 → モニタリング
Unit 2 を監視 → 切断 → Unit ID = 2 → 再接続 → モニタリング
```

---

### ケース②：複数ユニットを同時に表示する（例：Unit1 + Unit2 = 計8チャンネル）

**❌ 現状では非対応。大規模な改修が必要。**

| 変更項目 | 内容 |
|---|---|
| UI（UIR） | CH0〜CH7 の表示エリア・LED・フォルト表示の追加 |
| Strip Chart | 8トレース または 8分割グラフ |
| 通信ロジック | Unit1 と Unit2 を交互に読み取り（1サイクルで2回通信） |
| データ形式判定 | ユニットごとに DataFormat・信頼度を保持 |
| センサータイプ選択 | ユニットごとに設定が必要 |

## 実機 Web 設定の確認（2026-04-23）

CONPROSYS nano WEB Setting（機器情報画面）より確認。

**機器情報：**

| 項目 | 値 |
|---|---|
| ファームウェアバージョン | 1.61 |
| IP アドレス | 192.168.71.45 |
| 動作状態 | 動作中 |

**I/O モジュール搭載状況：**

| 物理スロット | 搭載モジュール | Unit ID（推定） |
|---|---|---|
| スロット 0 | CPSN-SSI-4C（現在） | 1（動作確認済み） |
| スロット 1 | −（空） | 2 |
| スロット 2 | −（空） | 3 |
| スロット 3 | −（空） | 4 |

スロット番号と Unit ID の対応は「スロット番号 + 1 = Unit ID」と推定。  
現在スロット 0 に挿入・Unit ID = 1 で動作確認済みのため整合性あり。

モジュールをスロット 1 に差し替えた場合、UI の `Unit ID = 2` に変更するだけで対応可能と推定。  
**差し替え前に Web 設定の「Modbus データ表示」画面でスロットと Unit ID の対応を確認することを推奨。**

## Modbus データ表示画面の確認（2026-04-23）

CONPROSYS nano WEB Setting「Modbus データ表示」画面より確認。

### スロット・レジスタ構成

| スロット | I/O モジュール | レジスタ種別 | 開始アドレス | 個数 |
|---|---|---|---|---|
| 0 | CPSN-SSI-4C | 入力レジスタ | 0 | 8 |
| 1〜3 | −（空き） | − | − | − |

**プログラムの固定値（startAddr=0, quantity=8）と完全一致。**

### スロット番号 ↔ Unit ID 対応（確定）

| 物理スロット | Unit ID | 備考 |
|---|---|---|
| スロット 0 | 1 | 動作確認済み・Modbus データ表示で確定 |
| スロット 1 | 2 | − |
| スロット 2 | 3 | − |
| スロット 3 | 4 | − |

### 確認時の生データ（Raw SSI 形式）

| CH | 生データ（16進） | 温度換算 |
|---|---|---|
| CH0 | 0x01006294 | ≈ 24.6℃ |
| CH1 | 0x010063F8 | ≈ 25.0℃ |
| CH2 | 0x0100648B | ≈ 25.1℃ |
| CH3 | 0x010065B7 | ≈ 25.4℃ |

上位バイト `0x01`（有効ビット D24=1）が全チャンネルで確認でき、  
データ形式判定「Raw SSI・信頼度高 4/4ch」とも整合している。

---

## 結論

| 用途 | 対応 | 備考 |
|---|---|---|
| 単一ユニット監視（ユニット切り替え運用） | ✅ 対応済み | Unit ID を変更して再接続 |
| 複数ユニット同時監視（8CH以上） | ❌ 未対応 | 大規模改修が必要 |
