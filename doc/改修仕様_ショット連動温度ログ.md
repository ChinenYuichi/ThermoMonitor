# ThermoMonitor 改修仕様：ショット連動温度ログ機能

作成日: 2026-05-01  
対象バージョン: v2.8.0 → v2.9.0  
設計分析書: `ショット連動温度ログ機能_設計分析.md`

---

## 機能概要

実験ネットワークの `time.txt` を監視し、ショット開始・終了を自動検出して
各チャンネルの **開始時温度** と **ショット中ピーク温度** をログに記録する。

```
[SHOT START] 14:32:05  CH0=23.4℃  CH1=22.8℃  CH2=22.5℃  CH3=22.6℃
[SHOT STOP ] 14:32:18  CH0=41.2℃  CH1=31.7℃  CH2=22.6℃  CH3=22.5℃  (peak)
```

**time.txt 判定ルール**

| time.txt値 | 意味 |
|---|---|
| `t < 0` | カウントダウン中 または 待機中 |
| `t ≥ 0` | ショット中（カウントアップ） |
| `t: 負→正` | **SHOT START**（通電開始） |
| `t: 正→負` | **SHOT STOP**（通電終了） |

### ② `FetchShotTime()` 関数の追加

**追加場所：** `ApplyChYAxisScaling()` の直前

**役割：** 実験ネットワークの `time.txt` をHTTP GETで取得してfloat値を返す。  
取得失敗時は `-9999.0` を返す（呼び出し元でスキップ判定する）。

```c
/*============================================================================
 * FetchShotTime - 実験ネットワークから time.txt を取得
 *
 * 取得成功時: time.txt の数値（秒）を返す
 * 取得失敗時: -9999.0 を返す
 *============================================================================*/
static double FetchShotTime(void)
{
    int    handle    = 0;
    char   req[256];
    char   buf[512];
    int    bytesRead = 0;
    double tVal      = -9999.0;
    char  *body;

    sprintf(req,
            "GET /expinfo/time.txt HTTP/1.0\r\n"
            "Host: csv02.exp.triam.kyushu-u.ac.jp\r\n"
            "Connection: close\r\n"
            "\r\n");

    if (ConnectToTCPServer(&handle, 80,
                           "csv02.exp.triam.kyushu-u.ac.jp",
                           NULL, NULL, 100) < 0)   /* タイムアウト 100ms */
        return -9999.0;

    if (SendTCPData(handle, req, (int)strlen(req), 100) < 0)
        goto cleanup;

    memset(buf, 0, sizeof(buf));
    ReadTCPData(handle, buf, (int)sizeof(buf) - 1, 100);

    /* HTTP レスポンスボディ（空行以降）を取得 */
    body = strstr(buf, "\r\n\r\n");
    if (body) {
        body += 4;
        tVal = atof(body);
    }

cleanup:
    DisconnectFromTCPServer(handle);
    return tVal;
}
```

> **注意：** `tcpsupp.h` を使用する。プロジェクトに未追加の場合は  
> CVI メニュー → `Edit` → `Add File to Project` → `tcpsupp.h` を追加すること。  
> また、インクルードに `#include <tcpsupp.h>` を追記すること。

### ③ `UpdateShotState()` 関数の追加

**追加場所：** `FetchShotTime()` の直後（`ApplyChYAxisScaling()` の直前）

```c
/*============================================================================
 * UpdateShotState - ショット状態を更新してSTART/STOPをログ出力
 *============================================================================*/
static void UpdateShotState(void)
{
    double    tNow;
    time_t    now;
    struct tm *tm_info;
    char      timeStr[32];
    int       ch;

    tNow = FetchShotTime();
    if (tNow <= -9000.0) return;   /* 取得失敗はスキップ */

    now     = time(NULL);
    tm_info = localtime(&now);
    strftime(timeStr, sizeof(timeStr), "%H:%M:%S", tm_info);

    if (g_prevShotTime < 0.0 && tNow >= 0.0) {
        /* ─── SHOT START 検出 ─── */
        for (ch = 0; ch < SSI4C_NUM_CHANNELS; ch++)
            g_shotPeakTemp[ch] = g_lastTemp[ch];   /* ピーク初期化 */

        LogMessageFmt("[SHOT START] %s  CH0=%.1f\xe2\x84\x83  CH1=%.1f\xe2\x84\x83"
                      "  CH2=%.1f\xe2\x84\x83  CH3=%.1f\xe2\x84\x83",
                      timeStr,
                      g_lastTemp[0], g_lastTemp[1],
                      g_lastTemp[2], g_lastTemp[3]);
        g_shotActive = 1;

    } else if (g_shotActive && tNow >= 0.0) {
        /* ─── ショット中：ピーク温度を更新 ─── */
        for (ch = 0; ch < SSI4C_NUM_CHANNELS; ch++)
            if (g_lastTemp[ch] > g_shotPeakTemp[ch])
                g_shotPeakTemp[ch] = g_lastTemp[ch];

    } else if (g_shotActive && tNow < 0.0) {
        /* ─── SHOT STOP 検出 ─── */
        LogMessageFmt("[SHOT STOP ] %s  CH0=%.1f\xe2\x84\x83  CH1=%.1f\xe2\x84\x83"
                      "  CH2=%.1f\xe2\x84\x83  CH3=%.1f\xe2\x84\x83  (peak)",
                      timeStr,
                      g_shotPeakTemp[0], g_shotPeakTemp[1],
                      g_shotPeakTemp[2], g_shotPeakTemp[3]);
        g_shotActive = 0;
    }

    g_prevShotTime = tNow;
}
```

> **補足：** `℃` をログに出力する場合、CVI の文字コードに応じて  
> `"℃"` をそのまま使うか、`\xb0\x43`（CP932）等に置き換えること。  
> ビルド環境に合わせて調整する。

## 動作確認手順

1. ビルドが通ることを確認（警告含めてチェック）
2. 実験ネットワーク接続環境で起動
3. `FetchShotTime()` が `-9999.0` を返さないことをログで確認  
   （デバッグ用に一時的に取得値をログ出力してもよい）
4. ショット待機中 → 開始 → 終了の遷移で以下がログに出力されること：
   ```
   [SHOT START] HH:MM:SS  CH0=xx.x℃ ...
   [SHOT STOP ] HH:MM:SS  CH0=xx.x℃ ...  (peak)
   ```
5. 実験ネットワーク未接続環境でも温度モニタが正常動作すること（`FetchShotTime` のエラーが無視されること）

## 注意事項

| 項目 | 内容 |
|---|---|
| タイムアウト | `ConnectToTCPServer` / `SendTCPData` / `ReadTCPData` のタイムアウトはすべて **100ms** 以下に設定すること。超過するとTimer_Callbackが詰まる |
| エラー処理 | ネットワーク未接続時は `FetchShotTime()` が `-9999.0` を返し `UpdateShotState()` が即リターンする。温度モニタへの影響なし |
| 文字コード | ログの `℃` 表記はCVIビルド環境の文字コードに合わせて調整する |
| デバウンス | time.txt が 0 前後で短時間に行き来する場合の誤検出が懸念される場合は、2回連続での状態変化を確認するデバウンス処理を追加すること |
