# ThermoMonitor 実機試験チェックリスト

## 1. 事前準備

- [ ] `ThermoMonitor.exe` が最新ビルドで生成されている
- [ ] PC と CPSN-MCB271 が同一ネットワークに接続されている
- [ ] 電源投入済み（CPSN-MCB271 / CPSN-SSI-4C）
- [ ] 熱電対（または疑似入力）が CH0-CH3 に配線済み
- [ ] 初期接続設定を確認  
  - IP: `10.1.1.101`（運用値に合わせる）  
  - Port: `502`  
  - Unit ID: `1`

---

## 2. 起動確認

- [ ] `ThermoMonitor.exe` を起動できる
- [ ] 画面表示に崩れや文字化けがない
- [ ] ログに起動メッセージが出る
- [ ] 初期状態が「未接続」になっている

合否基準:
- 起動エラーなしで画面が表示されること

---

## 3. 接続/切断確認

### 3-1. 接続
- [ ] `接続` ボタン押下で接続成功する
- [ ] ステータスが「接続中」に変わる
- [ ] `モニタリング開始` ボタンが有効化される
- [ ] ログに接続成功メッセージが出る

### 3-2. 切断
- [ ] `切断` ボタン押下で切断できる
- [ ] ステータスが「未接続」に戻る
- [ ] モニタリングボタンが無効化される
- [ ] ログに切断メッセージが出る

合否基準:
- 接続/切断がUI表示と一致すること

---

## 4. モニタリング基本動作

- [ ] `モニタリング開始` で定期更新が始まる
- [ ] 温度値が更新される（CH0-CH3）
- [ ] ログが継続的に更新される
- [ ] `モニタリング停止` で更新が止まる

合否基準:
- 開始・停止の操作で周期更新が制御できること

---

## 5. 表示仕様確認（色・フォルト）

- [ ] 正常時: 温度表示背景が正常色
- [ ] 軽微異常時: 警告色に変わる
- [ ] 異常時: フォルト色に変わる
- [ ] FLT LED とフォルトメッセージ表示が状態に一致する

合否基準:
- 温度背景色、FLT LED、メッセージが整合すること

---

## 6. 通信異常時の挙動

テスト例:
- LANケーブル抜去
- 対象機器電源OFF
- IP設定を一時的に誤値へ変更

確認項目:
- [ ] 読み取りエラー時にログへエラーが出る
- [ ] 必要時に自動停止/切断処理へ遷移する
- [ ] 再接続操作で復帰できる

合否基準:
- 異常時に無限ハングせず、オペレータが復帰可能であること

---

## 7. 長時間連続運転（任意）

- [ ] 30分以上の連続モニタリングで停止しない
- [ ] メモリ使用量が急増しない
- [ ] ログ表示が極端に遅くならない

合否基準:
- 連続運転で致命的不具合が発生しないこと
