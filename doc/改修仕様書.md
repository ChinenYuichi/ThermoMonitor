# ThermoMonitor 改修仕様書
## 自動認識とUIR連携による計測制御の最適化

**対象バージョン:** v2.1.3 → v2.2.0  
**作成日:** 2026-04-21  
**改修ファイル:** `modbus_tcp.h` / `modbus_tcp.c` / `ThermoMonitor_app.h` / `ThermoMonitor_ui.h` / `ThermoMonitor.c` / `ThermoMonitor.uir`

---

## 1. 改修目的

| 目的 | 内容 |
|---|---|
| 自動認識 | 接続直後に保持レジスタを読み取り、センサタイプ・データ形式を自動取得する |
| UIR連携 | 認識結果をUI上（専用ステータス表示）にリアルタイム反映する |
| 精度同期 | 除数（1000.0 または 1024.0）をデバイス設定から動的に取得し、温度誤差を解消する |
| 稼働安定化 | 連続エラーカウンタによるリトライロジックで長時間運用時の意図しない停止を防ぐ |

### 3.2 新規追加コントロール（UIRエディタで追加）

#### `PANEL_SENSOR_INFO` — センサ認識結果表示

接続後の自動認識結果（センサタイプ名・除数）を常時表示するテキストラベル。

| 属性 | 設定値 | 備考 |
|---|---|---|
| **Constant Name** | `SENSOR_INFO` | CVI が `PANEL_PANEL_SENSOR_INFO` として生成する |
| **コントロール型** | textMsg | 読み取り専用ラベル |
| **初期テキスト** | `認識待機中...` | 接続前の表示 |
| **配置位置** | `PANEL_STATUS` の直下 | 接続設定グループ内に配置 |
| **幅** | `PANEL_STATUS` と同幅 | レイアウト統一 |
| **Text Color** | デフォルト（白） | 認識後にコードで変更 |
| **Callback** | なし | |

**UIRエディタ操作手順:**
1. UIRエディタでパネルを開く
2. Create → Text Message でコントロールを追加
3. Constant Name に `SENSOR_INFO` を入力
4. テキストに `認識待機中...` を入力
5. `PANEL_STATUS` の直下に配置して保存
6. 保存後 `ThermoMonitor.h` が自動更新され `PANEL_PANEL_SENSOR_INFO` が追加される

> **CVI二重プレフィックス注意:** UIRエディタで `SENSOR_INFO` と設定すると `PANEL_PANEL_SENSOR_INFO` として生成される。`ThermoMonitor_ui.h` でエイリアスを定義して対処する（後述）。

## 5. `modbus_tcp.c` の変更

### 5.1 `Modbus_ReadHoldingRegisters` の実装追加

`Modbus_ReadInputRegisters`（FC 0x04）と構造が同一のため、
ファンクションコードのみ `FC_READ_HOLDING_REG`（0x03）に変更したコピーを実装する。

```c
/*============================================================================
 * Modbus_ReadHoldingRegisters - 保持レジスタ読み取り (FC 0x03)
 *============================================================================*/
int Modbus_ReadHoldingRegisters(int unitId, int startAddr, int quantity,
                                 uint16_t *regValues)
{
    uint8_t  reqBuf[MAX_ADU_SIZE];
    uint8_t  rspBuf[MAX_ADU_SIZE];
    int      byteCount, ret, i, expectedLen;
    uint16_t rxTransId, rxLength;

    if (!g_connected || !regValues || quantity < 1 || quantity > 125)
        return MODBUS_ERR_INVALID;

    g_transId++;
    PackUint16BE(&reqBuf[0], g_transId);
    PackUint16BE(&reqBuf[2], 0x0000);
    PackUint16BE(&reqBuf[4], 0x0006);

    reqBuf[6] = (uint8_t)unitId;
    reqBuf[7] = FC_READ_HOLDING_REG;           /* 0x03 */
    PackUint16BE(&reqBuf[8],  (uint16_t)startAddr);
    PackUint16BE(&reqBuf[10], (uint16_t)quantity);

    ret = ClientTCPWrite(g_tcpHandle, reqBuf, 12, RECV_TIMEOUT_MS);
    if (ret < 0) return MODBUS_ERR_SEND;

    ret = ClientTCPRead(g_tcpHandle, rspBuf, 6, RECV_TIMEOUT_MS);
    if (ret < 0)
        return (ret == kTCP_TimeOutErr) ? MODBUS_ERR_TIMEOUT : MODBUS_ERR_RECV;

    rxTransId = UnpackUint16BE(&rspBuf[0]);
    rxLength  = UnpackUint16BE(&rspBuf[4]);
    if (rxTransId != g_transId) return MODBUS_ERR_INVALID;

    ret = ClientTCPRead(g_tcpHandle, &rspBuf[6], (int)rxLength, RECV_TIMEOUT_MS);
    if (ret < 0)
        return (ret == kTCP_TimeOutErr) ? MODBUS_ERR_TIMEOUT : MODBUS_ERR_RECV;

    if (rspBuf[7] & 0x80) return MODBUS_ERR_EXCEPTION;

    byteCount   = rspBuf[8];
    expectedLen = quantity * 2;
    if (byteCount != expectedLen) return MODBUS_ERR_INVALID;

    for (i = 0; i < quantity; i++)
        regValues[i] = UnpackUint16BE(&rspBuf[9 + i * 2]);

    return MODBUS_OK;
}
```

### 8.2 `AutoRecognizeDeviceSettings` の実装追加

```c
/*============================================================================
 * AutoRecognizeDeviceSettings - 保持レジスタからデバイス設定を自動取得
 *
 * 接続成功直後に1回だけ呼び出す。
 * PANEL_UNIT_ID の現在値を使用して保持レジスタを読み取り、
 * g_currentSensorType / g_currentDivisor を更新する。
 * 結果は PANEL_SENSOR_INFO と LOG に反映する。
 *
 * 戻り値: MODBUS_OK=成功 / その他=失敗（デフォルト値で継続）
 *============================================================================*/
int AutoRecognizeDeviceSettings(void)
{
    uint16_t settings[3];   /* [0]=センサタイプ, [1]=予約, [2]=データ形式 */
    double   dUnitId;
    int      unitId;
    int      ret;
    int      sType;
    double   divisor;
    char     infoMsg[128];

    if (GetNumericCtrlAsDouble(panelHandle, PANEL_UNIT_ID, &dUnitId) < 0)
        dUnitId = DEFAULT_UNIT_ID;
    unitId = (int)dUnitId;

    /* 保持レジスタ 0x00〜0x02 を3レジスタ読み取り */
    ret = Modbus_ReadHoldingRegisters(unitId, ADDR_SENSOR_TYPE, 3, settings);
    if (ret != MODBUS_OK) {
        LogMessageFmt("自動認識失敗: %s (デフォルト設定で継続)", Modbus_ErrorString(ret));
        tryAttr(SetCtrlAttribute(panelHandle, PANEL_SENSOR_INFO,
                                 ATTR_CTRL_VAL, "認識失敗 (デフォルト: K型/1024)"));
        return ret;
    }

    sType   = (int)settings[0];
    /* データ形式レジスタ: 0x02 = 0.001℃単位 → 除数1000, それ以外 → 除数1024 */
    divisor = (settings[2] == 0x02) ? 1000.0 : 1024.0;

    /* 範囲チェック */
    if (sType < 0 || sType >= SENSOR_TYPE_COUNT)
        sType = SENSOR_TYPE_K;

    g_currentSensorType = sType;
    g_currentDivisor    = divisor;

    /* PANEL_SENSOR_INFO に認識結果を表示 */
    Fmt(infoMsg, "%s[%s] 除数: %.0f",
        SensorTypeNames[sType],
        (divisor == 1000.0) ? "0.001℃" : "標準",
        divisor);
    tryAttr(SetCtrlAttribute(panelHandle, PANEL_SENSOR_INFO,
                             ATTR_CTRL_VAL, infoMsg));
    tryAttr(SetCtrlAttribute(panelHandle, PANEL_SENSOR_INFO,
                             ATTR_TEXT_COLOR, 0x00008000));   /* 緑 */

    LogMessageFmt("自動認識完了: %s / 除数=%.0f", SensorTypeNames[sType], divisor);
    return MODBUS_OK;
}
```

### 8.4 `ReadAndDisplayTemperatures` の変更（リトライロジックと動的除数）

```c
/* 変更前 */
void ReadAndDisplayTemperatures(void)
{
    ChannelData chData[SSI4C_NUM_CHANNELS];
    /* ... */
    ret = SSI4C_ReadAllChannels(unitId, 0, chData);

    if (ret != MODBUS_OK) {
        LogMessageFmt("読み取りエラー: %s", Modbus_ErrorString(ret));
        if (ret == MODBUS_ERR_CONNECT || ret == MODBUS_ERR_RECV ||
            ret == MODBUS_ERR_TIMEOUT) {
            StopMonitoring();
            Modbus_Disconnect();
            UpdateConnectionStatus(0);
        }
        return;
    }
    /* ... */
}

/* 変更後 */
#define MAX_CONSECUTIVE_ERRORS  3   /* この回数連続でエラーが発生したら切断 */

void ReadAndDisplayTemperatures(void)
{
    ChannelData chData[SSI4C_NUM_CHANNELS];
    double      dUnitId;
    int         unitId;
    int         ret;
    int         ch;

    if (!Modbus_IsConnected()) {
        StopMonitoring();
        UpdateConnectionStatus(0);
        LogMessage("エラー: 接続が切断されました。");
        return;
    }

    if (GetNumericCtrlAsDouble(panelHandle, PANEL_UNIT_ID, &dUnitId) < 0)
        dUnitId = DEFAULT_UNIT_ID;
    unitId = (int)dUnitId;

    /* g_currentDivisor を用いて解析（自動認識した除数を適用） */
    ret = SSI4C_ReadAllChannels(unitId, 0, chData, g_currentDivisor);

    if (ret != MODBUS_OK) {
        g_consecutiveErrors++;
        LogMessageFmt("通信エラー(%d/%d): %s",
                      g_consecutiveErrors, MAX_CONSECUTIVE_ERRORS,
                      Modbus_ErrorString(ret));
        if (g_consecutiveErrors >= MAX_CONSECUTIVE_ERRORS) {
            LogMessage("連続エラー上限に達しました。切断します。");
            StopMonitoring();
            Modbus_Disconnect();
            UpdateConnectionStatus(0);
            tryAttr(SetCtrlAttribute(panelHandle, PANEL_SENSOR_INFO,
                                     ATTR_CTRL_VAL, "認識待機中..."));
            tryAttr(SetCtrlAttribute(panelHandle, PANEL_SENSOR_INFO,
                                     ATTR_TEXT_COLOR, 0x00808080));
        }
        return;
    }

    g_consecutiveErrors = 0;   /* 成功時はカウンタリセット */

    for (ch = 0; ch < SSI4C_NUM_CHANNELS; ch++)
        UpdateChannelDisplay(ch, &chData[ch]);

    UpdateStripChart(chData);
    g_readCount++;
}
```

## 9. グラフY軸スケール自動調整について（実装見送り）

提案されたセンサタイプ別Y軸自動スケールは **Strip Chart には適用不可** のため今回は見送る。

| 試みた方法 | 結果 | 備考 |
|---|---|---|
| `SetAxisScalingMode(... VAL_LEFT_YAXIS ...)` | 失敗 | Graph専用関数 |
| `ATTR_YSCALING` | -10エラー | Strip Chartに非対応 |
| `ATTR_YMIN` / `ATTR_YMAX` | 未定義 | CVI 2020に存在しない |

**代替:** `AutoRecognizeDeviceSettings` 内でセンサタイプ別の推奨Y軸レンジをログに出力する。

```c
/* AutoRecognizeDeviceSettings の末尾に追加 */
{
    double yMin = -100.0, yMax = 1400.0;
    if (sType == SENSOR_TYPE_R || sType == SENSOR_TYPE_S) { yMin = 0.0; yMax = 1800.0; }
    else if (sType == SENSOR_TYPE_T)                        { yMin = -100.0; yMax = 400.0; }
    else if (sType == SENSOR_TYPE_E)                        { yMin = -100.0; yMax = 1000.0; }
    LogMessageFmt("推奨グラフレンジ: %.0f〜%.0f ℃", yMin, yMax);
}
```

> 将来的にグラフを `CTRL_GRAPH`（固定長スクロール方式）に変更すれば、  
> `SetAxisScalingMode` による自動スケールが実現可能になる。

| `ADDR_SENSOR_TYPE` / `ADDR_DATA_FORMAT` | 実機の仕様書で確認後に確定すること |
| 自動認識の失敗 | 失敗時はデフォルト値（K型 / 除数1024）で継続。ログにエラー表示。 |
| 認識タイミング | 接続ボタン押下直後の1回のみ。モニタリング中は再認識しない。 |
| `g_consecutiveErrors` スコープ | `static` ではなくファイルスコープ変数として宣言（`StopMonitoring` や他の関数からリセットできるようにする） |
| `MAX_CONSECUTIVE_ERRORS` の定義位置 | `ThermoMonitor.c` 内 `ReadAndDisplayTemperatures` の直前に `#define` として定義 |
