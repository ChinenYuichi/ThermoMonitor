## 2. 起動と終了

### 起動

`ThermoMonitor.exe` をダブルクリックして起動する。

起動時に以下が自動設定される：

| 項目 | デフォルト値 |
|---|---|
| IP アドレス | `192.168.52.248` |
| ポート | `502` |
| ユニット ID | `1` |
| 更新間隔 | `500 ms` |
| 表示時間幅 | `60 秒` |
| センサータイプ | `K型`（Y軸レンジ -100〜1400℃） |
| 全チャンネル チェックボックス | ON |
| ショットCSVログ | OFF |
| ウィンドウ | 最大化 |

起動直後はログに以下が表示される：
```
[HH:MM:SS] CPSN-SSI-4C 熱電対温度モニタ v3.0.2 起動
[HH:MM:SS] 接続設定を確認して [接続] ボタンを押してください。
```

![起動直後の画面](起動直後.jpg)
*▲ 起動直後の画面。未接続状態・センサータイプ K型・全チャンネル表示 ON。*

### 終了

ウィンドウ右上の **×** ボタンをクリック。  
モニタリング中・CSV記録中の場合は自動的に停止・クローズ・切断してから終了する。

## 3. 接続設定

### コントロール一覧

| コントロール   | 内容                       | デフォルト値      |
|---------------|---------------------------|------------------|
| IP アドレス    | CPSN-MCB271 の IP アドレス | `192.168.52.248` |
| ポート         | Modbus TCP ポート番号      | `502`            |
| ユニット ID    | Modbus ユニット ID         | `1`              |
| 接続/切断ボタン | 接続・切断をトグル         | —                |
| ステータス表示  | 接続状態テキスト           | 「未接続」        |

### 接続手順

1. IP アドレス・ポート・ユニット ID を確認する
2. **[接続]** ボタンをクリックする
3. ステータス表示が **「接続中」**（緑文字）に変わることを確認する
4. ログにデータ形式の自動判定結果が表示される
5. **[モニタリング開始/停止]** ボタンが現れる

### 切断手順

1. モニタリング中の場合は先に **[モニタリング停止]** をクリックする
2. **[切断]** ボタンをクリックする
3. ステータス表示が **「未接続」**（グレー文字）に戻る

> **注意:** 接続失敗時はログにエラー内容が表示される。  
> IP アドレス・ネットワーク接続・機器の電源を確認すること。

![接続・判定後の画面](接続・判定後.jpg)
*▲ 接続成功後の画面。データ形式が自動判定され「Raw SSI (測定値) [自動判定]」・信頼度「高 (4/4ch一致)」が表示される。*

## 4. データ形式の自動判定

接続成功直後に FC04 生データのビットパターンを解析し、デバイスの Modbus データ形式を自動判定する。

### 判定される形式

| 形式 | 内容 |
|---|---|
| Raw SSI（測定値） | D24=有効ビット、D23-D0=温度÷1024 |
| 0.1℃+ステータス | reg0=温度×10、reg1=フォルトフラグ |
| 0.001℃ | int32÷1000（フォルトビットなし） |

### 表示コントロール

| コントロール   | 表示内容                     | 例                                        |
|---------------|-----------------------------|-------------------------------------------|
| データ形式情報 | 判定した形式名・判定方法       | 「データ形式: Raw SSI (測定値) [自動判定]」 |
| 信頼度表示     | 判定の確かさ・一致チャンネル数 | 「信頼度: 高 (4/4ch一致)」                 |

### 信頼度レベル

| 信頼度 | 一致チャンネル数     | 文字色  |
|-------|---------------------|--------|
| 高    | 3〜4ch              | 赤      |
| 中    | 1〜2ch              | 黄色    |
| 低    | 0ch（デフォルト適用） | グレー  |

> **注意:** デバイスの Web 設定でデータ形式を変更した場合は、再接続して判定をやり直すこと。

## 5. センサータイプの設定

センサータイプは Modbus で取得できないため、デバイスの Web 設定と合わせて**手動で設定**する。

### 設定手順

1. デバイスの Web 設定で設定されているセンサータイプを確認する
2. **センサータイプ選択リング** から同じ型を選択する
3. 全チャンネルのグラフ Y 軸レンジが自動的に切り替わる

### センサータイプ別 Y軸レンジ（初期値）

| センサータイプ   | Y軸 下限 | Y軸 上限 |
|----------------|---------|----------|
| J型            | -100℃   | 1200℃   |
| K型（デフォルト）| -100℃   | 1400℃   |
| E型            | -100℃   | 1000℃   |
| N型            | -100℃   | 1300℃   |
| R型            | 0℃      | 1800℃   |
| S型            | 0℃      | 1800℃   |
| T型            | -100℃   | 400℃    |
| Q型            | 0℃      | 150℃    |

> **注意:** センサータイプの変更後、Y軸レンジは各チャンネルの Ymax/Ymin コントロールにも反映される。  
> 個別に変更したい場合は次節「グラフY軸スケーリング」を参照。

> センサータイプの上下限超過の判定はデバイス側が行う。  
> プログラムのセンサータイプ設定はグラフ Y 軸レンジの変更のみに影響する。

## 6. グラフY軸スケーリング

各チャンネル（CH0〜CH3）の Y 軸レンジを個別に設定できる。

### コントロール一覧

| コントロール | 内容 | 初期値（センサータイプ依存） |
|---|---|---|
| CH0 Ymax | CH0 グラフ Y 軸上限（℃） | センサータイプの上限値 |
| CH0 Ymin | CH0 グラフ Y 軸下限（℃） | センサータイプの下限値 |
| CH1〜CH3 同様 | — | — |

### 操作方法

1. 変更したいチャンネルの **Ymax** または **Ymin** の値を入力する
2. 次の更新タイミング（最大 500ms）でグラフ Y 軸が自動的に更新される
3. Ymin ≥ Ymax の場合は無効として無視される

> センサータイプを変更すると、全チャンネルの Ymax/Ymin がそのセンサーの推奨レンジに  
> リセットされる。個別設定は再度入力し直すこと。

## 7. モニタリング操作

### コントロール一覧

| コントロール              | 内容                     | 設定範囲       |
|--------------------------|--------------------------|---------------|
| 更新間隔                  | データ取得・画面更新の間隔 | 任意（ms単位） |
| モニタリング開始/停止ボタン | モニタリングをトグル      | —             |

### 開始手順

1. 接続済みであることを確認する
2. **更新間隔**（ms）を設定する（デフォルト: 500 ms）
3. **[モニタリング開始]** ボタンをクリックする
4. 接続確認のためのテスト読み取りが実行される
5. ログに「モニタリング開始（更新間隔: XX ms）」が表示される
6. ボタンラベルが **「モニタリング停止」** に変わる

> モニタリング中は **接続/切断ボタン** と **更新間隔** がグレーアウトされ変更不可になる。

![モニタリング中の画面](モニタリング中.jpg)
*▲ モニタリング中の画面。各チャンネルの温度が更新され、ストリップチャートにトレースが描画される。*

### 停止手順

1. **[モニタリング停止]** ボタンをクリックする
2. 記録中の CSV ファイルが自動的にクローズされる
3. ログに「モニタリング停止（総読み取り回数: XX）」が表示される

### 連続エラー時の自動再接続

通信エラーが **3回連続** で発生した場合、自動再接続シーケンスに移行する。  
詳細は [13. 自動再接続機能](#13-自動再接続機能) を参照。

## 8. グラフ表示

### ストリップチャート

CH0〜CH3 の4チャンネルを縦4分割で表示する。

| チャンネル | トレース色 |
|-----------|-----------|
| CH0       | 赤        |
| CH1       | 黄緑       |
| CH2       | シアン     |
| CH3       | マゼンタ   |

グラフ背景色は黒。各チャンネル名（CH0〜CH3）をグラフ右上に色付きで表示する。

### グラフ設定コントロール

| コントロール              | 内容                              | デフォルト |
|--------------------------|---------------------------------|-----------|
| 表示時間幅                | グラフの横軸（時間幅）の設定（秒）   | 60秒      |
| CH0〜CH3 チェックボックス | 各チャンネルのグラフ表示 ON/OFF    | 全ON      |
| グラフクリアボタン        | 全チャンネルのグラフデータを消去     | —         |

### チャンネル表示 ON/OFF

- チェックボックスを **OFF** にするとそのチャンネルのトレースが非表示になる
- データは引き続き取得されるため、他チャンネルとのスクロール同期は維持される
- チェックを **ON** に戻すと元のトレース色で再表示される

### マルチモニター対応

ウィンドウを別の解像度のモニターへドラッグすると、全コントロールが自動的にスケーリングされる。

### ハードフォルト時のグラフ

ハードフォルト発生時はそのチャンネルのグラフが **-999.0℃** に急落し、視覚的に異常を明示する。

## 10. ショット連動温度ログ

実験ネットワーク上のサーバー（`csv02.exp.triam.kyushu-u.ac.jp`）から SHOT 状態を取得し、  
SHOT START / STOP を自動検出してログに記録する機能。

### 動作の流れ

モニタリング中、更新ごとに実験ネットワークサーバーへ HTTP GET を発行して `time.txt` を取得する。

```
取得値 ≥ 0.0 → SHOT 実施中
取得値 < 0.0 → SHOT 停止中
取得失敗    → 判定スキップ（前回状態を維持）
```

### ログ出力仕様

SHOT START 検出時:
```
[SHOT START] HH:MM:SS  CH0=XXX.X℃  CH1=XXX.X℃  CH2=XXX.X℃  CH3=XXX.X℃
```

SHOT STOP 検出時（STOP 時点の現在温度）:
```
[SHOT STOP ] HH:MM:SS  CH0=XXX.X℃  CH1=XXX.X℃  CH2=XXX.X℃  CH3=XXX.X℃
```

SHOT STOP 後、温度が安定（5秒間更新なし）した時点でピーク温度を出力:
```
[PEAK TIME ] CH0=XXX.X℃@HH:MM:SS  CH1=XXX.X℃@HH:MM:SS  ...
```

> **[PEAK TIME]** は SHOT STOP 後も温度が上昇し続けるケースに対応するため、  
> 温度が下降に転じて 5 秒間ピークが更新されなくなった時点で出力される。

### 使用条件

- 実験ネットワーク（TRIAM ネットワーク）に接続されていること
- モニタリング中であること（モニタリング停止中は検出しない）

## 11. ショットCSVログ

SHOT START 検出と連動して、温度データを CSV ファイルに自動記録する機能。

### 有効化方法

**「ショットCSVログ」チェックボックス** を ON にする。  
（デフォルト: OFF）

### CSV ファイルの仕様

| 項目 | 内容 |
|---|---|
| 保存先 | プログラムフォルダ（`ThermoMonitor.exe` と同じフォルダ） |
| ファイル名 | `ThermoMonitor_shot_YYYYMMDD_HHMMSS.csv` |
| 記録間隔 | 500ms（更新ごと） |
| 文字コード | UTF-8 BOM 付き（Excel で開いても文字化けなし） |
| フォーマット | `時刻,CH0(℃),CH1(℃),CH2(℃),CH3(℃),マーカー` |

### CSVファイルのサンプル

```csv
時刻,CH0(℃),CH1(℃),CH2(℃),CH3(℃),マーカー
11:11:00,25.2,25.3,25.2,25.0,SHOT START
11:11:00,26.1,25.3,25.2,25.0,
11:11:01,45.3,25.3,25.2,25.0,
...（500ms毎に記録）
11:11:52,124.7,25.4,25.3,25.1,SHOT STOP
11:11:52,121.2,25.4,25.3,25.1,
...（温度下降まで継続記録）
```

### ファイルの開閉タイミング

| タイミング | 動作 |
|---|---|
| SHOT START 検出 | 既存ファイルをクローズ → 新規 CSV を作成・開始 |
| SHOT STOP 検出 | ファイルはクローズせず記録継続 |
| 次の SHOT START | 前回ファイルをクローズ → 新規 CSV を作成 |
| チェックボックス OFF | 記録中ファイルをクローズ |
| モニタリング停止 | 記録中ファイルをクローズ |

### Excel でのグラフ化

保存された CSV を Excel で開き、時刻と温度データを選択してグラフを作成することで  
ショットごとの温度推移を可視化できる。

## 12. ログ表示

画面下部のテキストボックスに操作履歴・通信状態・エラー情報を時刻付きで記録する。  
ログは `C:\Users\quest\ThermoMonitor_log.txt` にも同時書き込みされる。

### ログフォーマット

```
[HH:MM:SS] メッセージ
```

### 主なログメッセージ

| メッセージ                                       | 内容                   |
|-------------------------------------------------|------------------------|
| `接続中... IP: X.X.X.X, Port: XXX`             | 接続試行中              |
| `データ形式判定: ○○ (信頼度: 高, 4/4ch)`         | データ形式の自動判定結果  |
| `接続成功: X.X.X.X:XXX`                         | 接続確立                |
| `接続失敗: エラー内容`                           | 接続失敗とその原因       |
| `モニタリング開始 (更新間隔: XXX ms)`            | モニタリング開始         |
| `モニタリング停止 (総読み取り回数: XX)`           | モニタリング停止         |
| `センサタイプ: ○型 / グラフレンジ: XX〜XX℃`      | センサータイプ変更       |
| `通信エラー(X/3): エラー内容`                    | 通信エラー発生           |
| `連続エラー上限に達しました。`                    | 自動再接続開始           |
| `自動再接続を開始します (5秒間隔・最大10回)。`    | 自動再接続シーケンス開始  |
| `自動再接続試行 (X/10)...`                       | 再接続試行中             |
| `自動再接続成功。モニタリングを再開します。`       | 再接続・モニタリング再開  |
| `[SHOT START] HH:MM:SS  CH0=...`               | SHOT 開始検出            |
| `[SHOT STOP ] HH:MM:SS  CH0=...`               | SHOT 停止検出            |
| `[PEAK TIME ] CH0=...@HH:MM:SS  ...`           | ピーク温度確定           |
| `CSVログ開始: ファイルパス`                      | CSV ファイル作成         |
| `CSVログ終了: ファイルパス`                      | CSV ファイルクローズ     |
| `切断しました。`                                 | 手動切断                 |

### ログクリア

**[ログクリア]** ボタンをクリックすると画面のログを全消去する。  
（`ThermoMonitor_log.txt` への書き込みは継続される）

> ログは新しいメッセージが追加されるたびに自動的に最新行までスクロールする。

## 13. 自動再接続機能

RDP 接続断・ネットワーク瞬断などで通信が切れた場合に自動で再接続を試みる機能。

### トリガー条件

以下のいずれかで自動再接続シーケンスを開始する：

- 通信エラーが **3回連続** で発生した場合
- `Modbus_IsConnected()` が切断を検知した場合

### 再接続の仕様

| 項目 | 値 |
|---|---|
| 再接続試行間隔 | 5 秒 |
| 最大試行回数 | 10 回 |
| 再接続成功時 | データ形式再判定 → モニタリング自動再開 |
| 10回失敗時 | 手動での [接続] 操作を促すメッセージを表示 |

### 自動再接続中の状態

- ステータス表示: 「自動再接続待機中...」（黄文字）
- **[接続]** ボタンをクリックすると自動再接続を中止して手動接続に移行できる

## 14. フォルト発生時の動作

### フォルト検出の仕組み

フォルトの判定はデバイス（CPSN-SSI-4C）側が行い、プログラムはその結果（フォルトビット）を受け取って表示する。

```
デバイスがフォルト検出
    ↓
フォルトビットを Modbus レジスタにセット
    ↓
プログラムがビットを解析
    ↓
LED 点滅・フォルトメッセージ・グラフで表示
```

### フォルト発生時の対処手順

1. フォルトメッセージでフォルト種別を確認する
2. **ソフトフォルト**（黄）の場合
   - センサー上下限超過であれば温度が正常範囲に戻るまで待機
   - CJ 異常であればデバイス周囲の環境温度を確認する
3. **ハードフォルト**（赤）の場合
   - 熱電対の配線・接続を確認する
   - デバイスの電源を確認する
   - 異常が解消されると LED 消灯・「正常」表示に戻る

### フォルト解消時の自動復帰

フォルト状態が解消されると、**次の更新タイミングで自動的に正常表示へ復帰**する。  
手動リセット操作は不要。

| フォルト種別 | 解消条件 | 復帰タイミング |
|---|---|---|
| ソフトフォルト | 温度が正常範囲に戻る | 次の更新（例: 500ms後） |
| ハードフォルト | 断線修復・ハード異常の解消 | 次の更新（例: 500ms後） |

> プログラムはデバイスのフォルトビットを毎回新たに読み直すため、  
> フォルト履歴の保持や手動リセットの仕組みはない。  
> デバイスの状態をリアルタイムでそのまま反映する設計。

## 15. 標準的な操作手順

### 通常の温度モニタリング

```
① ThermoMonitor.exe を起動する
    ↓
② センサータイプをデバイスの Web 設定と合わせて選択する
    ↓
③ IP アドレス・ポート・ユニット ID を確認する
    ↓
④ [接続] ボタンをクリックする
    → ログでデータ形式の自動判定結果を確認する
    ↓
⑤ 更新間隔を設定する（デフォルト 500 ms）
    ↓
⑥ [モニタリング開始] ボタンをクリックする
    → 各チャンネルの温度・グラフが更新されることを確認する
    ↓
⑦ 測定終了時は [モニタリング停止] → [切断] の順にクリックする
    ↓
⑧ ウィンドウを閉じる
```

### ショット連動 CSV ログを使用する場合

```
① 〜 ⑥ は上記「通常の温度モニタリング」と同じ
    ↓
⑦ 「ショットCSVログ」チェックボックスを ON にする
    ↓
⑧ SHOT START が検出されると自動的に CSV ファイルが作成される
    → ログに「CSVログ開始: ファイルパス」が表示される
    ↓
⑨ SHOT STOP 後もデータ記録は継続される
    → [PEAK TIME] がログに出力されるまで待機すると、
       真のピーク温度を確認できる
    ↓
⑩ 測定終了時は [モニタリング停止] → [切断] の順にクリックする
    → CSV ファイルが自動クローズされる
    ↓
⑪ プログラムフォルダ内の CSV ファイルを Excel で開いて解析する
```
