# ThermoMonitor 本番環境確認手順書

対象バージョン：v2.9.3  
作成日：2026-05-05

---

## 概要

実験ネットワーク上の実機（Modbusサーバー・ショット時刻サーバー）に接続し、ThermoMonitorの正常動作を確認する手順。

| 項目 | 値 |
|---|---|
| Modbusサーバー（温度計） | `192.168.52.248 : 502` |
| ショット時刻サーバー | `http://csv02.exp.triam.kyushu-u.ac.jp/expinfo/time.txt` |
| ログファイル | `C:\Users\quest\ThermoMonitor_log.txt` |

---

## 事前確認

### 1. ネットワーク接続確認

PowerShellで以下を実行し、Modbusサーバーへの疎通を確認する。

```powershell
ping 192.168.52.248
```

正常時の出力例：

```
192.168.52.248 からの応答: バイト数 =32 時間 =1ms TTL=255
```

疎通できない場合は、実験ネットワークへのLAN接続を確認すること。

### 2. hostsファイルの確認

ローカルテスト時に追記したエントリが残っていないことを確認する。

```powershell
Select-String -Path "C:\Windows\System32\drivers\etc\hosts" -Pattern "csv02"
```

**何も表示されなければOK。**  
もし表示された場合は、テキストエディタ（管理者権限）で `hosts` ファイルを開き、以下の行を削除する。

```
127.0.0.1 csv02.exp.triam.kyushu-u.ac.jp
```

## 起動・接続手順

### 1. ThermoMonitor を起動

`ThermoMonitor.exe` を起動する。

### 2. 接続設定の確認

起動後、以下のデフォルト値が設定されていることを確認する。

| 設定項目 | 値 |
|---|---|
| IPアドレス | `192.168.52.248` |
| ポート番号 | `502` |

### 3. 接続開始

「接続」ボタンを押し、ログに以下が表示されることを確認する。

```
接続中... IP: 192.168.52.248, Port: 502
接続成功: 192.168.52.248:502
```

接続失敗時はログのエラーメッセージを確認し、ネットワーク接続を再チェックする。

---

## 動作確認項目

### 確認1：温度データの取得

接続後、4チャンネルの温度グラフが更新されていることを目視確認する。

- ストリップチャートの波形が動いている
- 異常値（-9999℃など）が連続して表示されていない

### 確認2：ショット連動ログの確認（v2.9.3）

実ショット発生時に、以下のログが正しく記録されることを確認する。

```powershell
Get-Content -Path C:\Users\quest\ThermoMonitor_log.txt -Tail 20
```

**期待されるログ出力：**

```
[SHOT START] Shot time: X.X s, Start temp CH0: XX.X ℃
[SHOT STOP]  Shot time: -5.0 s, Peak temp CH0: XX.X ℃
```

### 確認3：-12 エラーが出ないこと（v2.9.3 修正内容）

ログに以下のエラーが**出ないこと**を確認する。

```
NON-FATAL RUN-TIME ERROR: "DisconnectFromTCPServer" ... Error Code: -12
```

v2.9.3 以前では、HTTP/1.0 の `Connection: close` によりサーバーが先にTCP切断するため、このエラーが発生していた。v2.9.3 で修正済み。

---

## 自動再接続の確認（v2.5.0以降）

RDP接続切断やネットワーク一時断が発生した場合、自動再接続が機能することを確認する。

- 再接続間隔：5秒
- 最大試行回数：10回
- ログに `再接続試行中...` → `接続成功` が記録される

---

## 異常時の対応

| 症状 | 確認・対処 |
|---|---|
| 接続失敗が続く | `ping 192.168.52.248` で疎通確認。LANケーブル・スイッチを確認 |
| 温度が -9999℃ 表示 | センサーの配線・Modbusユニットの電源を確認 |
| SHOT START が検出されない | `hosts` ファイルにローカルテスト用エントリが残っていないか確認 |
| SHOT START の検出が遅い | v2.9.2 で修正済み。バージョンを確認すること |
| ログファイルが更新されない | `C:\Users\quest\ThermoMonitor_log.txt` のパスと書き込み権限を確認 |

---

## 確認完了後

正常動作が確認できたら、通常のモニタリング運用へ移行する。

- ログファイルは追記形式のため、必要に応じて定期的にバックアップすること
- ログファイルの場所：`C:\Users\quest\ThermoMonitor_log.txt`
