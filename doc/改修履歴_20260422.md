# ThermoMonitor 改修履歴

改修日: 2026-04-22

---

## 1. データ形式名称の文字化け修正（modbus_tcp.c）

**問題:** `DataFormatNames[]` の2・3番目エントリが EUC-JP エスケープシーケンスで記述されており、Windows（Shift-JIS）環境で文字化けしていた。

**修正:** エスケープシーケンスを直接日本語表記に変更。

```c
/* 修正前 */
"0.1\xa3\xa9+\xa5\xb9\xa5\xc6\xa1\xbc\xa5\xbf\xa5\xb9"
"0.001\xa3\xa9"

/* 修正後 */
"0.1℃+ステータス"
"0.001℃"
```

---

## 2. Strip Chart Y軸レンジのセンサータイプ連動（ThermoMonitor.c）

**追加機能:** センサータイプ選択時に `SetAxisScalingMode` で全チャンネルのY軸レンジを自動変更。

**判明事項（CVI 2020）:**
- `VAL_YAXIS` は未定義（Graph用）
- Strip Chart には **`VAL_LEFT_YAXIS`** を使用すること

```c
SetAxisScalingMode(panelHandle, g_stripHandle[ch],
                   VAL_LEFT_YAXIS, VAL_MANUAL, yMin, yMax);
```

---

## 3. センサータイプ・Y軸レンジのデフォルト設定（ThermoMonitor.c）

**追加機能:** 起動時にセンサータイプを J型・Y軸レンジを -100〜1200℃ にデフォルト設定。

- `ApplyDefaultCtrlValues`: `PANEL_SENSOR_TYPE_SEL` を `SENSOR_TYPE_J` に設定
- `InitStripChart`: 全チャンネルY軸レンジを -100〜1200℃ で初期化

---

## 4. データ形式ヒューリスティック判定の誤判定修正（modbus_tcp.c）

**問題:** `TEMP_01` 形式で温度が **25.6〜51.1℃** の場合、reg0 上位バイトが `0x01` となり Raw SSI と誤判定されていた。

**修正:** Raw SSI 判定条件を強化。

```c
/* 修正前 */
if (upperByte == 0x01) → Raw SSI

/* 修正後 */
if (upperByte == 0x01 && (regLow >> 8) != 0x00) → Raw SSI
/* TEMP_01 のフォルトレジスタ(reg1)は上位バイトが常に 0x00 のため区別可能 */
```

---

## 5. switch文 warning 修正（modbus_tcp.c）

**問題:** `SSI4C_ParseData` の switch 文で `DATA_FORMAT_RAW_SSI` ケースに `default:` が重複しており warning が発生。

**修正:** `default:` ラベルを削除。

---

## 6. ログ自動スクロール（ThermoMonitor.c）

**追加機能:** ログ追加時に最新行へ自動スクロール。

```c
GetNumTextBoxLines(panelHandle, PANEL_LOG, &numLines);
SetCtrlAttribute(panelHandle, PANEL_LOG, ATTR_FIRST_VISIBLE_LINE, numLines);
```

---

## 7. ログ最大行数の設定

**判明事項:** `ATTR_MAX_LINES` は CVI 2020 で未定義。

**対処:** UIRエディタで `PANEL_LOG` の **`Max # of Lines`** プロパティを直接設定する。

---

## 8. チェックボックスOFF時のグラフ非表示（ThermoMonitor.c）

**変更前:** チェックボックスOFF時に前回値でフラットなグラフが引かれ続けた。

**変更後:** チェックOFF時はトレース色を黒（背景色）に変更して非表示。再チェック時は元の色に復元。データは同期維持のため継続プロット。

```c
SetTraceAttribute(panelHandle, g_stripHandle[ch], 1,
                  ATTR_TRACE_COLOR,
                  chkVal ? (int)g_traceColors[ch] : 0x00000000);
```

---

## 9. Strip Chart ラベルフォントサイズ変更（ThermoMonitor.c）

**判明事項（CVI 2020）:**
- `ATTR_LABEL_HEIGHT` はラベルエリアの高さでフォントサイズではない
- フォントサイズは **`ATTR_LABEL_POINT_SIZE`** で設定可能

```c
SetCtrlAttribute(panelHandle, g_stripHandle[ch], ATTR_LABEL_POINT_SIZE, 20);
```

---

## 10. LEDフォルト色分け（ThermoMonitor.c）

**変更前:** フォルト種別に関わらず LED は ON/OFF のみ。

**変更後:** `ATTR_ON_COLOR` でフォルト種別ごとに色分け。

| 状態 | LED |
|---|---|
| 正常 | 消灯 |
| ソフトフォルト（センサ上下限超過等） | 黄色点灯 |
| ハードフォルト・データ無効 | 赤点灯 |

---

## 11. ハードフォルト時グラフ表示（ThermoMonitor.c）

**変更前:** ハードフォルト時は前回値でフラットなグラフが継続。

**変更後:** ハードフォルト時は `-999.0℃`（`FAULT_ERROR_TEMP`）をプロットして視覚的に異常を明示。

---

## 12. PANEL_FAULT_CHx のフォルトメッセージ表示修正（ThermoMonitor.c）

**問題:** `ATTR_CTRL_VAL` では文字列コントロールへの設定が効かない場合があった。

**修正:** `SetCtrlVal` に変更。

```c
/* 修正前 */
tryAttr(SetCtrlAttribute(panelHandle, ctrlFault[ch], ATTR_CTRL_VAL, data->faultMsg));

/* 修正後 */
SetCtrlVal(panelHandle, ctrlFault[ch], data->faultMsg);
```

---

## 13. フォルト時 LED・PANEL_FAULT_CHx 点滅（ThermoMonitor.c）

**追加機能:** フォルト発生時に LED とフォルトメッセージテキストを点滅表示。

**実装方法:** 既存タイマー（`Timer_Callback`）を利用してトグル変数を切り替え。

```c
/* Timer_Callback 内 */
g_blinkState = !g_blinkState;

/* UpdateChannelDisplay 内 */
SetCtrlVal(panelHandle, ctrlLED[ch], g_blinkState);          /* LED 点滅 */
ATTR_TEXT_COLOR = g_blinkState ? COLOR_FAULT : 0x00000000;   /* 文字点滅 */
```

**点滅速度:** モニタリング間隔と同じ（500ms設定時 = 1Hz）

## フォルト表示仕様まとめ

| 状態 | LED | PANEL_FAULT_CHx テキスト | PANEL_FAULT_CHx 色 | グラフ |
|---|---|---|---|---|
| 正常 | 消灯 | 「正常」 | 緑（固定） | 実測値 |
| ソフトFLT | 黄色点滅 | フォルトメッセージ | 黄点滅 | 前回値フラット |
| ハードFLT | 赤点滅 | フォルトメッセージ | 赤点滅 | -999.0℃ |

## 実機フォルト発生時の動作フロー

デバッグコード削除後も、実機でのフォルト（断線・上下限超過等）発生時には
同じ LED 点滅・フォルトメッセージ表示が動作する。

```
デバイス（CPSN-SSI-4C）がフォルトを検出
    ↓
フォルトビット（D24-D31）を Modbus レジスタにセット
    ↓
SSI4C_ParseData() がフォルトビットを解析
    hasHardFault / hasSoftFault / faultMsg をセット
    ↓
UpdateChannelDisplay() が同じ処理で表示
    ├─ LED 点滅（赤: ハードFLT / 黄: ソフトFLT）
    ├─ PANEL_FAULT_CHx テキスト点滅（同色）
    ├─ PANEL_TEMP_CHx 背景色変化
    └─ Strip Chart に -999.0℃ プロット（ハードFLT時）
```

### 実機で発生するフォルト種別

| フォルト | 種別 | 主な原因 |
|---|---|---|
| センサハードFLT | ハード | 熱電対の断線・短絡 |
| ADCハードFLT | ハード | ADC 回路異常 |
| CJハードFLT | ハード | 冷接点回路異常 |
| センサ上限超過 | ソフト | 測定温度がセンサ上限を超過 |
| センサ下限超過 | ソフト | 測定温度がセンサ下限を下回る |
| CJソフトFLT | ソフト | 冷接点温度が許容範囲外 |
| ADC範囲外 | ソフト | ADC 入力が測定範囲外 |

※ フォルト判定の上下限はデバイスの Web 設定のセンサータイプに基づく。
　 プログラム側の `PANEL_SENSOR_TYPE_SEL` はグラフY軸レンジのみに影響する。

---

## フォルト解消時の自動復帰仕様

フォルト状態が解消されると、次の更新タイミングで**自動的に正常表示へ復帰**する。  
手動リセット操作は不要。

| フォルト種別 | 解消条件 | 復帰タイミング |
|---|---|---|
| ソフトフォルト | 温度が正常範囲に戻る | 次の更新（例: 500ms後） |
| ハードフォルト | 断線修復・ハード異常の解消 | 次の更新（例: 500ms後） |

**根拠:** `SSI4C_ParseData()` はデバイスのフォルトビットを毎回新たに読み直して  
`hasHardFault` / `hasSoftFault` を設定するため、前回のフォルト状態を保持しない。  
プログラムはデバイスの状態をリアルタイムでそのまま反映する設計。
