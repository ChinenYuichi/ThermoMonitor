## modbus_tcp.c

Modbus TCP 通信ライブラリ。CPSN-MCB271 / CPSN-SSI-4C との通信処理を担当する。

---

### 公開関数

#### `Modbus_Connect`
```c
int Modbus_Connect(const char *ipAddr, int port, int timeoutMs)
```
| 項目 | 内容 |
|---|---|
| 役割 | 指定 IP・ポートへ Modbus TCP 接続を確立する |
| 引数 | `ipAddr`: 接続先IPアドレス文字列 / `port`: ポート番号（通常502） / `timeoutMs`: タイムアウト[ms]（0以下でデフォルト3000ms） |
| 戻り値 | `MODBUS_OK`（0）: 成功 / `MODBUS_ERR_CONNECT`（-1）: 接続失敗 |
| 備考 | 既に接続中の場合は `Modbus_Disconnect` を呼んでから再接続する |

---

#### `Modbus_Disconnect`
```c
int Modbus_Disconnect(void)
```
| 項目 | 内容 |
|---|---|
| 役割 | Modbus TCP 接続を切断し、TCPハンドルを解放する |
| 引数 | なし |
| 戻り値 | 常に `MODBUS_OK`（0） |
| 備考 | 未接続状態で呼んでも安全（何もしない） |

---

int Modbus_IsConnected(void)

| 項目 | 内容 |
|---|---|

#### `Modbus_ReadInputRegisters`
```c
int Modbus_ReadInputRegisters(int unitId, int startAddr, int quantity, uint16_t *regValues)
```
| 項目 | 内容 |
|---|---|
| 役割 | Function Code 0x04（入力レジスタ読み取り）でレジスタ値を取得する |
| 引数 | `unitId`: Modbus ユニットID / `startAddr`: 開始レジスタアドレス / `quantity`: 読み取りレジスタ数（1〜125） / `regValues`: 結果格納バッファ |
| 戻り値 | `MODBUS_OK`: 成功 / `MODBUS_ERR_SEND`: 送信失敗 / `MODBUS_ERR_RECV`: 受信失敗 / `MODBUS_ERR_TIMEOUT`: タイムアウト / `MODBUS_ERR_EXCEPTION`: Modbus例外 / `MODBUS_ERR_INVALID`: データ不正 |
| 備考 | MBAP ヘッダのトランザクションIDで応答整合性を確認する |

---

#### `SSI4C_DetectDataFormat`
```c
DataFormat SSI4C_DetectDataFormat(int unitId, int startAddr, int *confidence)
```
| 項目 | 内容 |
|---|---|
| 役割 | FC04 生データのビットパターンからデータ形式を自動判定する（ヒューリスティック） |
| 引数 | `unitId`: Modbus ユニットID / `startAddr`: 開始レジスタアドレス / `confidence`: 判定一致チャンネル数の出力先（0〜4、NULL可） |
| 戻り値 | `DATA_FORMAT_RAW_SSI` / `DATA_FORMAT_TEMP_01` / `DATA_FORMAT_TEMP_001` のいずれか |
| 判定基準 | Raw SSI: 上位バイト=0x01 / 0.001℃: 上位16bit≤0x001B / 0.1℃+ST: reg1上位バイト=0x00 |
| 備考 | 全CH投票方式で最多票の形式を選択。読取失敗時は `DATA_FORMAT_RAW_SSI` を返す |

---

#### `SSI4C_ReadAllChannels`
```c
int SSI4C_ReadAllChannels(int unitId, int startAddr, ChannelData *chData, DataFormat fmt)
```
| 項目 | 内容 |
|---|---|
| 役割 | SSI-4C の全4チャンネルを一括読み取りし、`ChannelData` 構造体に解析結果を格納する |
| 引数 | `unitId`: Modbus ユニットID / `startAddr`: 開始レジスタアドレス / `chData`: 結果格納配列（4要素以上） / `fmt`: データ形式 |
| 戻り値 | `MODBUS_OK`: 成功 / その他: 通信エラーコード |
| 備考 | 内部で `Modbus_ReadInputRegisters` → `SSI4C_ParseData` の順に処理する |

---

#### `SSI4C_ParseData`
```c
void SSI4C_ParseData(uint16_t rawHigh, uint16_t rawLow, ChannelData *chData, DataFormat fmt)
```
| 項目 | 内容 |
|---|---|
| 役割 | レジスタペア（上位16bit / 下位16bit）を指定データ形式に従って解析し、温度・フォルト情報を取得する |
| 引数 | `rawHigh`: 上位レジスタ値 / `rawLow`: 下位レジスタ値 / `chData`: 解析結果の出力先 / `fmt`: データ形式 |
| 戻り値 | なし |
| データ形式別処理 | **RAW_SSI**: D31-D24=フォルトバイト, D23-D0=温度÷1024 / **TEMP_01**: reg0=温度×10(0.1℃), reg1下位バイト=フォルトフラグ / **TEMP_001**: int32÷1000(0.001℃) |

---

#### `SSI4C_GetFaultMessage`
```c
void SSI4C_GetFaultMessage(uint8_t faultByte, char *msgBuf, int bufLen)
```
| 項目 | 内容 |
|---|---|
| 役割 | フォルトバイトを解析して日本語フォルトメッセージ文字列を生成する（主に Raw SSI 形式用） |
| 引数 | `faultByte`: フォルトバイト（D31-D24） / `msgBuf`: 出力バッファ / `bufLen`: バッファサイズ |
| 戻り値 | なし |
| 出力例 | 正常時: `"正常"` / 異常時: `"センサハードFLT"` `"ADCハードFLT"` 等の組み合わせ |

---

const char* Modbus_ErrorString(int errCode)

---

### メイン関数

#### `main`
```c
int main(int argc, char *argv[])
```
| 項目 | 内容 |
|---|---|
| 役割 | アプリケーション起動・初期化・UIイベントループ実行 |

#### `InitStripChart`
```c
static void InitStripChart(void)
```
| 項目 | 内容 |
|---|---|
| 役割 | UIRのストリップチャートプレースホルダを削除し、CH0〜CH3 の4つのストリップチャートをプログラムで生成・初期設定する |
| 生成処理 | UIRの配置情報取得 → `DiscardCtrl` → `NewCtrl(CTRL_STRIP_CHART)` × 4 |
| 設定内容 | 縦4等分配置・ラベル色・黒背景（`ATTR_PLOT_BGCOLOR`）・トレース色（`SetTraceAttribute`）・X軸スケール |
| 備考 | CVI 2020 のバグでUIRにてトレース数を保存できないためプログラム生成方式を採用 |

---

#### `ApplyTimeRange`
```c
static void ApplyTimeRange(void)
```
| 項目 | 内容 |
|---|---|
| 役割 | `PANEL_TIME_RANGE`[秒] と `PANEL_INTERVAL`[ms] の設定値から、ストリップチャートの X軸スケール（`ATTR_XAXIS_GAIN`）と表示点数（`ATTR_POINTS_PER_SCREEN`）を計算・設定する |
| 表示点数範囲 | 最小10点〜最大7200点 |
| 備考 | `StartMonitoring` とモニタリング開始時に呼ばれる |

---

#### `UpdateStripChart`
```c
static void UpdateStripChart(const ChannelData *chData)
```
| 項目 | 内容 |
|---|---|

| 備考 | チェックボックスOFFまたはフォルト発生チャンネルは前回値を維持し、全トレースのスクロールを同期させる |

### 公開関数（アプリケーション制御）

#### `DetectAndApplyDataFormat`
```c
int DetectAndApplyDataFormat(void)
```
| 項目 | 内容 |
|---|---|
| 役割 | 接続直後に `SSI4C_DetectDataFormat` を呼び、判定結果を `g_currentDataFormat` に格納してUIに反映する |

| 呼び出し元 | `BtnConnect_Callback`（接続成功時に1回のみ） |

---

#### `StartMonitoring`
```c
void StartMonitoring(void)
```
| 項目 | 内容 |
|---|---|
| 役割 | タイマーを有効化してモニタリングを開始する |
| 処理 | タイマー間隔更新 → タイマー有効化 → 接続・間隔コントロールをグレーアウト → ボタンラベル変更 → 初回即時取得 |
| 備考 | 未接続状態ではエラーログを出して早期リターンする |

---

#### `StopMonitoring`
```c
void StopMonitoring(void)
```
| 項目 | 内容 |
|---|---|
| 役割 | タイマーを無効化してモニタリングを停止する |
| 処理 | タイマー停止 → コントロールのグレーアウト解除 → ボタンラベル変更 → 総読み取り回数ログ出力 |
| 備考 | 既に停止中の場合は即リターン |

---

#### `ReadAndDisplayTemperatures`
```c
void ReadAndDisplayTemperatures(void)
```

### コールバック関数

| 関数名 | 対応コントロール | 処理概要 |
|---|---|---|
| `PanelCallback` | パネル本体 | `EVENT_CLOSE` でモニタリング停止・切断・終了 |
| `BtnConnect_Callback` | 接続/切断ボタン | 接続: `Modbus_Connect` → `DetectAndApplyDataFormat` / 切断: `Modbus_Disconnect` |
| `BtnMonitor_Callback` | モニタリング開始/停止ボタン | `StartMonitoring` / `StopMonitoring` をトグル |
| `BtnClear_Callback` | ログクリアボタン | `ResetTextBox` でログをクリア |
| `BtnGraphClear_Callback` | グラフクリアボタン | `ClearStripChart` で全チャートのデータを消去し前回値バッファをリセット |
| `ChkChannel_Callback` | チャネル表示チェックボックス | 処理なし（`UpdateStripChart` 内で参照） |
| `SensorTypeSel_Callback` | センサタイプ選択リング | Y軸レンジを `SetAxisScalingMode(VAL_LEFT_YAXIS, VAL_MANUAL, yMin, yMax)` で設定しログ出力 |
| `Timer_Callback` | タイマー | `EVENT_TIMER_TICK` で `ReadAndDisplayTemperatures` を呼び出す |
