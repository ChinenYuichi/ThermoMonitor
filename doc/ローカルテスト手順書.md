# ThermoMonitor ローカルテスト手順書

対象バージョン：v2.9.x（ショット連動温度ログ機能）  
作成日：2026-05-05

---

## 概要

実験ネットワーク（`csv02.exp.triam.kyushu-u.ac.jp`）に接続できない環境でも、ローカルPC上にPythonサーバーを立てることでショット連動温度ログ機能の動作確認が可能。

---

## 事前準備

### 1. テストファイルの作成

管理者権限のPowerShellで以下を実行する。

```powershell
New-Item -ItemType Directory -Force -Path C:\test_server\expinfo
Set-Content -Path C:\test_server\expinfo\time.txt -Value "-5.0"
```

### 2. Pythonサーバーの起動

**管理者権限のPowerShell**で以下を実行する（ポート80を使用するため管理者権限が必要）。

```powershell
python -m http.server 80 --directory C:\test_server
```

サーバーが起動すると以下のようなメッセージが表示される。

```
Serving HTTP on :: port 80 (http://[::]:80/) ...
```

### 3. hostsファイルの編集

`C:\Windows\System32\drivers\etc\hosts` をテキストエディタ（管理者権限）で開き、末尾に以下を追記する。

```
127.0.0.1 csv02.exp.triam.kyushu-u.ac.jp
```

これにより、ThermoMonitorからの接続要求がローカルのPythonサーバーに転送される。

---

## テスト実行手順

### 1. ThermoMonitor を起動

`ThermoMonitor.exe` を起動し、通常通りモニタリングを開始する。

### 2. ログファイルを準備

テスト中のログは以下のファイルに追記される。事前に内容を確認しておく。

```
C:\Users\quest\ThermoMonitor_log.txt
```

### 3. テストシーケンスの実行

`time.txt` の値をPowerShellで書き換えることで、ショットの開始・停止をシミュレートする。

#### ステップ1：待機状態の確認

```powershell
Set-Content -Path C:\test_server\expinfo\time.txt -Value "-5.0"
```

- 期待動作：ログに何も出力されない（待機中）

#### ステップ2：SHOT START の確認

```powershell
Set-Content -Path C:\test_server\expinfo\time.txt -Value "1.0"
```

- 期待動作：ログに `[SHOT START]` と開始時の温度が記録される

#### ステップ3：SHOT STOP の確認

```powershell
Set-Content -Path C:\test_server\expinfo\time.txt -Value "-5.0"
```

- 期待動作：ログに `[SHOT STOP]` とピーク温度が記録される

### 4. ログファイルの確認

```powershell
Get-Content -Path C:\Users\quest\ThermoMonitor_log.txt -Tail 20
```

正常動作時の出力例：

```
[2026-05-XX XX:XX:XX] Connected to Modbus server.
[2026-05-XX XX:XX:XX] [SHOT START] Shot time: 1.0 s, Start temp CH0: 25.3 ℃
[2026-05-XX XX:XX:XX] [SHOT STOP]  Shot time: -5.0 s, Peak temp CH0: 87.6 ℃
```

---

## テスト後の後片付け

**必ず実施すること。** 本番ネットワーク接続時に誤動作する原因となる。

### 1. Pythonサーバーの停止

PowerShellで `Ctrl+C` を押してサーバーを停止する。

### 2. hostsファイルの編集を元に戻す

`C:\Windows\System32\drivers\etc\hosts` から追記した以下の行を削除する。

```
127.0.0.1 csv02.exp.triam.kyushu-u.ac.jp
```

---

## 注意事項

- Pythonサーバーの起動は**管理者権限**が必要（ポート80使用のため）
- hostsファイルの編集も**管理者権限**が必要
- テスト後は必ずhostsファイルを元に戻すこと
- ログファイルへの書き込みは `C:\Users\quest\` 配下を使用すること（`C:\` 直下はWindows 11で管理者権限が必要なため）
