# ThermoMonitor ハイブリッドテスト手順書

対象バージョン：v2.9.3  
作成日：2026-05-05

---

## 概要

実験ショットを行わずに、本番環境で以下を同時に確認するテスト手順。

| 項目 | 接続先 | 内容 |
|---|---|---|
| 温度データ | `192.168.52.248:502`（実機） | 実際のセンサー温度を取得 |
| ショット時刻 | `127.0.0.1`（ローカル） | `time.txt` を手動書き換えでシミュレート |

hostsファイルでショット時刻サーバーのみをローカルに向けることで、実ショットなしにSHOT START/STOPの記録を確認できる。

---

## 事前準備

### 1. テストファイルの作成

```powershell
New-Item -ItemType Directory -Force -Path C:\test_server\expinfo
Set-Content -Path C:\test_server\expinfo\time.txt -Value "-36000.0"
```

### 2. Pythonサーバーの起動

**管理者権限のPowerShell**で実行する。

```powershell
python -m http.server 80 --directory C:\test_server
```

起動確認：

```
Serving HTTP on :: port 80 (http://[::]:80/) ...
```

### 3. hostsファイルの編集

`C:\Windows\System32\drivers\etc\hosts` をテキストエディタ（管理者権限）で開き、末尾に追記する。

```
127.0.0.1 csv02.exp.triam.kyushu-u.ac.jp
```

### 4. 設定確認

以下のコマンドでローカルへのリダイレクトと実機への疎通を確認する。

```powershell
# ショット時刻サーバー → ローカルに向いているか確認
Invoke-WebRequest -Uri 'http://csv02.exp.triam.kyushu-u.ac.jp/expinfo/time.txt' -UseBasicParsing | Select-Object -ExpandProperty Content

# Modbusサーバー → 実機に繋がるか確認
Test-NetConnection -ComputerName 192.168.52.248 -Port 502
```

期待値：
- `time.txt` の値が `-36000.0`（ローカルのファイルが返っている）
- `TcpTestSucceeded: True`

## テスト実行手順

### 1. ThermoMonitor を起動・接続

`ThermoMonitor.exe` を起動し、IPアドレス `192.168.52.248`・ポート `502` で接続する。

ログに以下が表示されることを確認する。

```
接続成功: 192.168.52.248:502
```

温度グラフ（4CH）が実際のセンサー値で更新されていることを目視確認する。

### 2. テストシーケンスの実行

`time.txt` を書き換えてSHOT START/STOPを発生させる。

#### ステップ1：待機状態の確認

```powershell
Set-Content -Path C:\test_server\expinfo\time.txt -Value "-36000.0"
```

期待動作：ログに何も出力されない（待機中）

#### ステップ2：SHOT START の確認

```powershell
Set-Content -Path C:\test_server\expinfo\time.txt -Value "1.0"
```

期待動作：ログに `[SHOT START]` と実機からの開始温度が記録される

#### ステップ3：SHOT STOP の確認（数十秒後）

```powershell
Set-Content -Path C:\test_server\expinfo\time.txt -Value "-36000.0"
```

期待動作：ログに `[SHOT STOP]` と実機からのピーク温度が記録される

### 3. ログの確認

```powershell
Get-Content -Path C:\Users\quest\ThermoMonitor_log.txt -Tail 20
```

正常時の出力例（実機温度が記録される）：

```
[SHOT START] Shot time: 1.0 s, Start temp CH0: 35.2 ℃
[SHOT STOP]  Shot time: -36000.0 s, Peak temp CH0: 36.8 ℃
```

### 4. -12 エラーが出ないことの確認

ログに以下が**出ないこと**を確認する。

```
NON-FATAL RUN-TIME ERROR: "DisconnectFromTCPServer" ... Error Code: -12
```

## テスト後の後片付け

**必ず実施すること。** 本番ショット時に誤動作する原因となる。

### 1. Pythonサーバーの停止

PowerShellで `Ctrl+C` を押す。

### 2. hostsファイルを元に戻す

`C:\Windows\System32\drivers\etc\hosts` から追記した行を削除する。

```
127.0.0.1 csv02.exp.triam.kyushu-u.ac.jp
```

削除後、以下のコマンドで確認する（何も表示されなければOK）。

```powershell
Select-String -Path "C:\Windows\System32\drivers\etc\hosts" -Pattern "csv02"
```

## ローカルテストとの違い

| 項目 | ローカルテスト | ハイブリッドテスト |
|---|---|---|
| 温度データ | ダミー（接続なし） | **実機から取得** |
| ショット時刻 | ローカル `time.txt` | ローカル `time.txt` |
| 目的 | ショットログ機能の動作確認 | **実温度＋ショットログの総合確認** |
