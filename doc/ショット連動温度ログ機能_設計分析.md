# ショット連動温度ログ機能 設計分析

**作成日：2026-05-01**  
**対象プログラム：ThermoMonitor v2.8.0 / Screen3.3r.py**

---

## 1. 概要

実験ネットワーク上の `time.txt` を監視し、ショット開始（t=0）・終了を自動検出して、
各チャンネルの **開始時温度** と **ショット中の最高（ピーク）温度** をログに記録する機能の設計分析。

---

## 2. 参照元：Screen3.3r.py の仕組み

`http://csv02.exp.triam.kyushu-u.ac.jp/expinfo/time.txt` から1秒毎に時刻値（秒単位）を取得し、
その符号と大きさで実験フェーズを判定している。

| time.txt値 | 実験フェーズ | 背景色 |
|---|---|---|
| `t < -1000` | 次ショットまで時間あり（待機） | 濃緑 |
| `-1000 ≤ t < -60` | カウントダウン60秒以上 | 赤 |
| `-60 < t < 0` | カウントダウン60秒以内（警告） | 黄 |
| `t ≥ 0` | ショット中（カウントアップ） | 青 |

---

## 3. ショット時間の流れ

```
time.txt値の推移：

... → -500 → -100 → -60 → -10 → 0 → 1 → 2 → ... → 13 → -1200 → ...
                                  ↑                   ↑
                             SHOT START           SHOT STOP
                            （t:負→正）           （t:正→負）
```

- **t = 0**：通電開始（ショット開始）
- **t > 0**：ショット進行中（カウントアップ）
- **t が再び負に戻る**：ショット終了

---

## 4. 実現可能性の結論

**実現可能。** 以下の理由による：

- ThermoMonitor の `g_lastTemp[]` は `ReadAndDisplayTemperatures()` が毎周期更新しており、常に最新温度が参照できる
- `LogMessageFmt()` によるログ出力機能が既に実装済み
- 標準Cライブラリの `time()` / `strftime()` で現在時刻が取得できる（`<ansi_c.h>` 既インクルード）
- CVI の TCP 関数（`tcpsupp.h`）で HTTP GET が実装できる

---

## 5. 処理の流れ

```
Timer_Callback（500ms毎）
  ├─ ReadAndDisplayTemperatures()   ← g_lastTemp[] を更新（既存）
  ├─ ApplyChYAxisScaling()          ← Y軸スケーリング（既存）
  └─ UpdateShotState()              ← 新規追加
        ├─ FetchShotTime()          time.txt を HTTP GET で取得・パース
        ├─ 前回値と比較して状態遷移を検出
        │    t: 負 → 正  → SHOT START ログ出力
        │    t: 正 → 負  → SHOT STOP  ログ出力（ピーク温度）
        └─ ショット中はg_shotPeakTemp[] をリアルタイム更新
```

---

## 6. 状態機械の設計

| 状態 | 条件 | 動作 |
|---|---|---|
| **待機** | `t < 0`、`g_shotActive = 0` | ピーク温度バッファを維持 |
| **START検出** | `前回 t < 0` かつ `今回 t ≥ 0` | 現在時刻・現在温度（全CH）をログ出力、ピーク温度を現在温度で初期化 |
| **ショット中** | `t ≥ 0`、`g_shotActive = 1` | 毎周期 `g_lastTemp[]` と `g_shotPeakTemp[]` を比較してピークを更新 |
| **STOP検出** | `前回 t ≥ 0` かつ `今回 t < 0` | 現在時刻・ピーク温度（全CH）をログ出力 |

---

## 7. 必要な追加要素

### 7-1. グローバル変数

```c
static int    g_shotActive            = 0;
static double g_prevShotTime          = -9999.0;
static double g_shotPeakTemp[SSI4C_NUM_CHANNELS];
```

### 7-2. 現在時刻の取得

標準Cライブラリを使用（`<ansi_c.h>` 既インクルード）。

```c
time_t    now      = time(NULL);
struct tm *tm_info = localtime(&now);
char      timeStr[32];
strftime(timeStr, sizeof(timeStr), "%H:%M:%S", tm_info);
/* → timeStr = "14:32:05" */
```

### 7-3. ピーク温度の追跡

```c
/* START 時：現在温度でピークを初期化 */
for (ch = 0; ch < SSI4C_NUM_CHANNELS; ch++)
    g_shotPeakTemp[ch] = g_lastTemp[ch];

/* ショット中（毎周期）：現在温度がピークを超えたら更新 */
for (ch = 0; ch < SSI4C_NUM_CHANNELS; ch++)
    if (g_lastTemp[ch] > g_shotPeakTemp[ch])
        g_shotPeakTemp[ch] = g_lastTemp[ch];

/* STOP 時：g_shotPeakTemp[] にショット中の最大温度が格納済み */
```

### 7-4. `FetchShotTime()` 関数の概要

CVI の TCP 関数（`tcpsupp.h`）を使い、HTTP GET リクエストを送信して `time.txt` の値を取得・パースする。

```c
/* 骨格 */
ConnectToTCPServer(&handle, 80, "csv02.exp.triam.kyushu-u.ac.jp", ...);
SendTCPData(handle, "GET /expinfo/time.txt HTTP/1.0\r\nHost:...\r\n\r\n", ...);
ReadTCPData(handle, buf, sizeof(buf), ...);
DisconnectFromTCPServer(handle);
/* レスポンスボディを float にパースして返す */
```

タイムアウトは **100ms 以下** に設定し、メインループをブロックしない。

### 7-5. `UpdateShotState()` 関数の概要

```c
static void UpdateShotState(void)
{
    double tNow;
    time_t now;
    struct tm *tm_info;
    char   timeStr[32];
    int    ch;

    tNow = FetchShotTime();         /* time.txt 取得 */
    if (tNow <= -9000.0) return;    /* 取得失敗時はスキップ */

    now     = time(NULL);
    tm_info = localtime(&now);
    strftime(timeStr, sizeof(timeStr), "%H:%M:%S", tm_info);

    if (g_prevShotTime < 0.0 && tNow >= 0.0) {
        /* START 検出 */
        for (ch = 0; ch < SSI4C_NUM_CHANNELS; ch++)
            g_shotPeakTemp[ch] = g_lastTemp[ch];
        LogMessageFmt("[SHOT START] %s  CH0=%.1f℃  CH1=%.1f℃  CH2=%.1f℃  CH3=%.1f℃",
                      timeStr,
                      g_lastTemp[0], g_lastTemp[1], g_lastTemp[2], g_lastTemp[3]);
        g_shotActive = 1;

    } else if (g_shotActive && tNow >= 0.0) {
        /* ショット中：ピーク更新 */
        for (ch = 0; ch < SSI4C_NUM_CHANNELS; ch++)
            if (g_lastTemp[ch] > g_shotPeakTemp[ch])
                g_shotPeakTemp[ch] = g_lastTemp[ch];

    } else if (g_shotActive && tNow < 0.0) {
        /* STOP 検出 */
        LogMessageFmt("[SHOT STOP ] %s  CH0=%.1f℃  CH1=%.1f℃  CH2=%.1f℃  CH3=%.1f℃  (peak)",
                      timeStr,
                      g_shotPeakTemp[0], g_shotPeakTemp[1],
                      g_shotPeakTemp[2], g_shotPeakTemp[3]);
        g_shotActive = 0;
    }

    g_prevShotTime = tNow;
}
```

## 9. 注意点・課題

| 項目 | 内容 | 対策 |
|---|---|---|
| **ネットワーク到達性** | 実験ネットワーク（csv02）に未接続の環境では取得失敗 | エラー時はスキップして継続（温度モニタには影響なし） |
| **タイムアウト** | HTTP 取得の遅延がタイマーコールバックを詰まらせる | タイムアウト 100ms 以下に設定 |
| **t=0 前後のノイズ** | 0 を短時間で行き来する可能性 | デバウンス（2回連続で状態変化を確認）を推奨 |
| **ピーク確定** | ショット終了（t が負に戻った時点）でピーク温度を出力するため、ショット中の最大値を確実に捉えられる | g_shotPeakTemp[] による運転中の逐次更新 |
