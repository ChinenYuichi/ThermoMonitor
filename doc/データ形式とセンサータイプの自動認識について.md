# データ形式とセンサータイプの自動認識について

作成: 2026-04-21  
対象: CPSN-SSI-4C / CPSN-MCB271 (Modbus TCP)

---

## 結論

| 項目 | Modbusレジスタに出るか | 自動判定 |
|---|---|---|
| データ形式（Raw SSI / 0.1℃ / 0.001℃） | 出る（ビットパターンの違いとして） | ✓ 可能 |
| センサータイプ（J/K/E/N/R/S/T型） | 出ない（内部処理で消費される） | ✗ 不可能 |

---

## データ形式 → 自動判定できる理由

FC04（入力レジスタ読み取り）で得た **生データのビットパターン自体** が、
Webで設定したデータ形式によって構造が異なるため、データを見るだけで判断できる。

### ヒューリスティック判定ルール（SSI4C_DetectDataFormat）

```
Raw SSI  : 上位バイト（D31-D24）= 0x01
           → D24=有効ビット(1)、D25-D31=フォルトなし(0) の正常時パターン

0.001℃  : 上位16bit（reg0）≤ 0x001B
           → 1800℃×1000=1,800,000=0x1B7740 より、正温度なら上位wordが小さい正数

0.1℃+ST : reg1の上位バイト = 0x00
           → ステータスバイトは下位バイトのみ使用、上位バイトは常に0x00
```

全4チャンネルで投票を行い、最多票の形式を採用（confidence = 一致チャンネル数）。

---

## センサータイプ → 自動判定できない理由

センサータイプ（J/K/E/N/R/S/T）は CPSN-SSI-4C の **内部設定値** であり、
Modbus レジスタには一切現れない。

```
デバイス内部処理:
  センサータイプ設定 → 熱電対rawデータ → 温度変換 → FC04レジスタ (温度℃)
                ↑                                        ↑
         Web設定のみ                            Modbusで読めるのはここだけ
```

FC04 で受け取るのは変換済みの「温度値 (℃)」のみ。
同じ500℃でも、K型センサーか J型センサーかはデータ上で区別できない。

センサータイプはデバイスの **Web設定画面でのみ** 確認・変更できる。

---

## v2.3.0 での対応方針

| 項目 | 対応 |
|---|---|
| データ形式 | 接続直後に `SSI4C_DetectDataFormat()` で自動判定し `PANEL_SENSOR_INFO` に表示 |
| センサータイプ | `PANEL_SENSOR_TYPE_SEL`（リングコントロール）でユーザーが手動設定 |

センサータイプ選択時は `SensorTypeSel_Callback` が呼ばれ、
選択したタイプに対応する **推奨グラフY軸レンジ** をログに出力する。

### 推奨グラフレンジ対応表

| センサータイプ | 推奨レンジ |
|---|---|
| J型 | -100 ～ 1200 ℃ |
| K型 | -100 ～ 1372 ℃（デフォルト） |
| E型 | -100 ～ 1000 ℃ |
| N型 | -100 ～ 1300 ℃ |
| R型 | 0 ～ 1768 ℃ |
| S型 | 0 ～ 1768 ℃ |
| T型 | -100 ～ 400 ℃ |

> **備考**: CVI 2020 の Strip Chart は Y軸自動スケールAPIが使用不可のため、
> 推奨レンジはログ出力のみとし、グラフへの自動反映は行わない。
