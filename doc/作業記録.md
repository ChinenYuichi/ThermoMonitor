## 対象ハードウェア

### CPU ユニット
- **型式:** CPSN-MCB271-S1-041 / CPSN-MCB271-1-041 / CPSN-MCB271-1-081
- **メーカー:** CONTEC CO., LTD.
- **デフォルトIPアドレス:** 10.1.1.101
- **通信方式:** Modbus TCP (port 502) / Modbus RTU (RS-232C)

### 熱電対モジュール
- **型式:** CPSN-SSI-4C
- **チャネル数:** 4チャネル (差動入力)
- **分解能:** 24-bit (1/1024 ℃)
- **変換速度:** 251ms / チャネル
- **コールドジャンクション補償:** 内蔵

| N | IEC60584-1 | -100 〜 1300 ℃ |

## Modbus TCP 通信仕様

### 接続設定

| 項目 | 値 |
|---|---|
| プロトコル | Modbus TCP |
| ポート番号 | 502 |
| 最大接続数 | 5 |
| ユニットID | 1 (デフォルト) |

### CPSN-SSI-4C レジスタマップ

**入力レジスタ (Function Code: 0x04)**

| アドレス | データ | サイズ |
|---|---|---|
| 0 | SSIデータ (CH0 上位16bit) | 16bit |
| 1 | SSIデータ (CH0 下位16bit) | 16bit |
| 2 | SSIデータ (CH1 上位16bit) | 16bit |
| 3 | SSIデータ (CH1 下位16bit) | 16bit |
| 4 | SSIデータ (CH2 上位16bit) | 16bit |
| 5 | SSIデータ (CH2 下位16bit) | 16bit |
| 6 | SSIデータ (CH3 上位16bit) | 16bit |
| 7 | SSIデータ (CH3 下位16bit) | 16bit |

> **注意:** 上位レジスタと下位レジスタは1回のコマンドで読み出すこと。

### SSI データ形式 (32bit)

```
D31  D30  D29  D28  D27  D26  D25  D24 | D23 .............. D0
 |    |    |    |    |    |    |    |         温度データ (24bit)
 |    |    |    |    |    |    |    +-- 有効ビット (1=有効)
 |    |    |    |    |    |    +------- ADC範囲外フォルト
 |    |    |    |    |    +------------ センサ下限超過フォルト
 |    |    |    |    +----------------- センサ上限超過フォルト
 |    |    |    +---------------------- CJソフトフォルト
 |    |    +--------------------------- CJハードフォルト
 |    +-------------------------------- ADCハードフォルト
 +------------------------------------- センサハードフォルト
```

### フォルトビット詳細

| ビット | フォルト名 | 出力結果 |
|---|---|---|
| D31 | センサハードフォルト (断線・ADC異常・CJセンサ異常) | -999℃ |
| D30 | ADCハードフォルト (大きな外部ノイズ等) | -999℃ |
| D29 | CJハードフォルト (冷接点センサエラー) | -999℃ |
| D28 | CJソフトフォルト (冷接点センサ通常範囲外) | 測定値異常 |
| D27 | センサ上限超過 | 測定値異常 |
| D26 | センサ下限超過 | 測定値異常 |
| D25 | ADC範囲外 | 測定値異常 |
| D24 | 有効ビット (0=無効→結果を破棄) | 有効(1)なら正常 |

### 温度変換式

```
温度 [℃] = 符号付き24bit整数 (D0-D23) / 1024
```

**変換例:**

| 温度 [℃] | 温度データ (hex) |
|---|---|
| 1024 | 100000h |
| 1 | 000400h |
| 1/1024 | 000001h |
| 0 | 000000h |
| -1/1024 | FFFFFFh |
| -1 | FFFC00h |
| -273.15 | FBBB67h |

### ファイル一覧

```
ThermoMonitor/
```

### モジュール構成

#### `modbus_tcp.c / modbus_tcp.h`
Modbus TCP 通信の低レベル実装。CVI の `tcpsupp.h` を使用。

| 関数 | 説明 |
|---|---|
| `Modbus_Connect()` | Modbus TCP サーバへ接続 |
| `Modbus_Disconnect()` | 切断 |
| `Modbus_IsConnected()` | 接続状態取得 |
| `Modbus_ReadInputRegisters()` | 入力レジスタ読み取り (FC 0x04) |
| `SSI4C_ReadAllChannels()` | 全チャネル温度一括読み取り |
| `SSI4C_ParseData()` | SSI データ解析 (温度・フォルト抽出) |
| `SSI4C_GetFaultMessage()` | フォルトメッセージ文字列生成 |

#### `ThermoMonitor.c / ThermoMonitor.h`
メインプログラム。プログラムによるUI構築 (UIRファイル不要)。

| 関数 | 説明 |
|---|---|
| `main()` | エントリポイント |
| `CreateUI()` | パネル・コントロールを動的に生成 |
| `BtnConnect_Callback()` | 接続/切断ボタン処理 |
| `BtnMonitor_Callback()` | モニタリング開始/停止ボタン処理 |
| `Timer_Callback()` | 定期読み取りタイマー処理 |
| `ReadAndDisplayTemperatures()` | 温度データ取得・画面更新 |
| `UpdateChannelDisplay()` | チャネル個別の表示更新 |
| `LogMessage()` | タイムスタンプ付きログ出力 |

## 使用手順

1. CPSN-MCB271 の電源を入れ、PCとLAN接続する
2. CPSN-MCB271 のIPアドレスを確認 (デフォルト: 10.1.1.101)
3. `ThermoMonitor.exe` を起動
4. IPアドレス・ポート番号を入力
5. **[接続]** ボタンをクリック
6. 接続成功後、**[モニタリング開始]** をクリック
7. 各チャネルの温度が更新間隔毎に表示される

## トラブルシューティング

### [2026-04-13] プロジェクトファイルを開けない

**現象:**
CVI2020 でプロジェクトを開いた際に以下のエラーダイアログが表示される。

```
Unable to open project file "c:\...\ThermoMonitor\ThermoMonitor.prj"
(Pathname is not in the platform independent format at line 4)
```

**原因:**
LabWindows/CVI のプロジェクトファイル (`.prj`) はパスを **プラットフォーム独立形式** で記述する必要がある。
Windows 形式 (`C:/path/...`) ではなく、CVI 形式 (`/C/path/...`) でなければならない。

**対処:**
`ThermoMonitor.prj` 内の全パス記述を以下のルールで修正した。

| 修正箇所 | 修正前 | 修正後 |
|---|---|---|
| `Pathname Line0001` | `"C:/Gyrotron_Program/..."` | `"/C/Gyrotron_Program/..."` |
| 各ファイルの `Path Line0001` (4箇所) | `"C:/..."` | `"/C/..."` |
| `Target File Line0001` (Debug/Release 各1箇所) | `"C:/..."` | `"/C/..."` |
| `Number of Files` | `3` | `4` (ヘッダファイル2件が未カウントだったため) |

**補足:**
CVI プロジェクトファイルのパス形式ルール:
- ドライブ文字 `C:` → `/C`
- 区切り文字はすべてスラッシュ `/`
- 例: `C:/Gyrotron_Program/foo.c` → `/C/Gyrotron_Program/foo.c`

### [2026-04-13] 実行時エラー (NON-FATAL / FATAL RUN-TIME ERROR)

**現象:**
```
NON-FATAL RUN-TIME ERROR: "ThermoMonitor.c", line 127 : Function SetCtrlAttributeUtf8: (return value == -13). Invalid control ID
NON-FATAL RUN-TIME ERROR: "ThermoMonitor.c", line 132 : Function SetCtrlAttributeUtf8: (return value == -13). Invalid control ID
NON-FATAL RUN-TIME ERROR: "ThermoMonitor.c", line 142 : Function SetCtrlAttributeUtf8: (return value == -13). Invalid control ID
FATAL RUN-TIME ERROR:     "ThermoMonitor.c", line 146 : Invalid argument type: found 'int', expected 'double'.
```

**原因①: 無効なコントロールID (-1)**

`NewCtrl()` の戻り値を保存せず `-1` をコントロールIDとして渡していた。  
LabWindows/CVI では `-1` は有効なコントロールIDではない。

```c
/* 修正前 (NG) */
NewCtrl(panel, CTRL_TEXT_MSG, "ラベル", top, left);
SetCtrlAttribute(panel, -1, ATTR_WIDTH, 200);   /* -1 は無効 */

/* 修正後 (OK) */
ctrlTmp = NewCtrl(panel, CTRL_TEXT_MSG, "ラベル", top, left);
SetCtrlAttribute(panel, ctrlTmp, ATTR_WIDTH, 200);
```

**対処①:** `CreateUI()` 関数内に `int ctrlTmp;` を追加し、ラベル・区切り線など使い捨てコントロール全箇所 (計9箇所) で `ctrlTmp` に戻り値を保存するよう修正。

**原因②: `ATTR_MIN_VALUE` / `ATTR_MAX_VALUE` の型不一致**

`SetCtrlAttribute()` で `ATTR_MIN_VALUE` / `ATTR_MAX_VALUE` に `int` リテラルを渡していたが、数値コントロールの場合これらは `double` 型を要求する。

```c
/* 修正前 (NG) */
SetCtrlAttribute(panel, ctrlPort, ATTR_MIN_VALUE, 1);       /* int → エラー */
SetCtrlAttribute(panel, ctrlPort, ATTR_MAX_VALUE, 65535);   /* int → エラー */

/* 修正後 (OK) */
SetCtrlAttribute(panel, ctrlPort, ATTR_MIN_VALUE, 1.0);
SetCtrlAttribute(panel, ctrlPort, ATTR_MAX_VALUE, 65535.0);
```

**対処②:** 数値コントロール3個 (ctrlPort / ctrlUnitID / ctrlInterval) の MIN/MAX 計6箇所を `double` リテラルに修正。

### [2026-04-13] 実行時エラー — SetCtrlVal の型不一致

**現象:**
```
FATAL RUN-TIME ERROR: "ThermoMonitor.c", line 149 : Invalid argument type: found 'int', expected 'double'.
```

**原因:**
`CTRL_NUMERIC` コントロールに対する `SetCtrlVal()` は `double` 型を要求するが、  
`DEFAULT_PORT` / `DEFAULT_UNIT_ID` / `DEFAULT_INTERVAL_MS` はすべて `int` 定数だった。

**対処:** 該当3箇所に `(double)` キャストを追加。

```c
/* 修正前 */
SetCtrlVal(panel, ctrlPort,     DEFAULT_PORT);        /* int → エラー */
SetCtrlVal(panel, ctrlUnitID,   DEFAULT_UNIT_ID);     /* int → エラー */
SetCtrlVal(panel, ctrlInterval, DEFAULT_INTERVAL_MS); /* int → エラー */

/* 修正後 */
SetCtrlVal(panel, ctrlPort,     (double)DEFAULT_PORT);
SetCtrlVal(panel, ctrlUnitID,   (double)DEFAULT_UNIT_ID);
SetCtrlVal(panel, ctrlInterval, (double)DEFAULT_INTERVAL_MS);
```

**補足:** `CTRL_STRING` / `CTRL_ROUND_LED` / `CTRL_TIMER` 等の他コントロールは型が異なるため対象外。

### [2026-04-13] 実行時エラー — コールバック関数の戻り値型不一致

**現象:**
```
FATAL RUN-TIME ERROR: "ThermoMonitor.c", line 168 :
  Invalid argument type: found 'pointer to void function(...)', expected '__cdecl int function(...)'.
```

**原因:**
CVI のコントロールコールバック (`CtrlCallbackPtr`) は `int` を返す関数ポインタ型だが、  
`BtnConnect_Callback` / `BtnMonitor_Callback` / `BtnClear_Callback` / `Timer_Callback` を `void` で宣言していた。

**対処:** 4つのコントロールコールバックを `void` → `int` に変更し、各関数末尾および早期リターン箇所に `return 0;` を追加。

```c
/* 修正前 */
void CVICALLBACK BtnConnect_Callback(...) { if (event != EVENT_COMMIT) return; ... }

/* 修正後 */
int  CVICALLBACK BtnConnect_Callback(...) { if (event != EVENT_COMMIT) return 0; ... return 0; }
```

**補足:** パネルコールバック (`PanelCallback`) は最初から `int` で定義していたため対象外。  
CVI のコールバック型まとめ:
- パネルコールバック: `int CVICALLBACK func(int panel, int event, void*, int, int)`
- コントロールコールバック: `int CVICALLBACK func(int panel, int ctrl, int event, void*, int, int)`
