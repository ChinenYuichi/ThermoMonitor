# Modbus 例外レスポンスについて

## 概要

Modbus デバイスが**リクエストを正常に処理できない**とき、通常レスポンスの代わりに返すエラー通知。

### レスポンスフォーマット

```
通常レスポンス:  FC = 0x04        （要求と同じファンクションコード）
例外レスポンス:  FC = 0x84        （0x04 | 0x80 = MSBを立てる）
                ExceptionCode    （理由コード 1バイト）
```

### プログラム内の判定箇所（modbus_tcp.c）

```c
if (rspBuf[7] & 0x80) return MODBUS_ERR_EXCEPTION;  // MSBが1なら例外
```

---

## 例外コードの種類

| コード | 意味 | 発生条件の例 |
|---|---|---|
| `0x01` | 不正ファンクションコード | FC04 以外を送った場合 |
| `0x02` | 不正データアドレス | 存在しないレジスタアドレスを指定 |
| `0x03` | 不正データ値 | 無効な読み取り数量（0 や 126以上など） |
| `0x04` | デバイス障害 | デバイス内部エラー |

---

## 本プログラムで例外が発生しにくい理由

本プログラムが送るリクエストは**固定値**であり、プログラム起動時から変化しない。

```c
// Modbus_ReadInputRegisters 内（固定値）
reqBuf[7] = FC_READ_INPUT_REG;          // FC = 0x04（固定）
PackUint16BE(&reqBuf[8],  0);           // 開始アドレス = 0（固定）
PackUint16BE(&reqBuf[10], 8);           // 数量 = 8レジスタ（固定）
```

| 項目 | 値 | 根拠 |
|---|---|---|
| ファンクションコード | `0x04`（FC04固定） | CPSN-SSI-4C が対応済み |
| 開始アドレス | `0`（固定） | 正規アドレス |
| 読み取り数量 | `8`（4ch × 2レジスタ） | Modbus 上限 125 以内 |

これらはハードコードされているため、**プログラム側からの誤ったリクエストによる例外は原理的に発生しない。**

---

## 例外が発生しうるケース

| ケース | 例外コード |
|---|---|
| CPSN-SSI-4C の Web 設定でレジスタマップを変更した | `0x02`（不正アドレス） |
| ファームウェアのバグでデバイス内部エラーが起きた | `0x04`（デバイス障害） |
| 将来的に読み取りアドレス・数量を UI で変更可能にした場合 | `0x02` / `0x03` |

---

## 現在の例外発生時の動作

`MODBUS_ERR_EXCEPTION` は連続エラーカウンタ（`g_consecutiveErrors`）に加算される。  
3回連続で発生した場合、通信エラーと同様に切断処理が実行される。

```
例外レスポンス受信
  → MODBUS_ERR_EXCEPTION 返却
  → g_consecutiveErrors++
  → 3回到達 → StopMonitoring() / Modbus_Disconnect()
```

運用上の例外発生リスクはほぼゼロのため、現時点では例外を切断カウントから除外する対応は不要と判断。
