## v2.9.4  26.05.05

### 修正内容：SHOT STOP 未検出バグ修正

**ファイル：** `ThermoMonitor.c`

**症状：**
- 本番サーバー（`csv02.exp.triam.kyushu-u.ac.jp`）接続時に SHOT STOP が検出されない

**原因：**
- `UpdateShotState()` の早期リターン条件 `if (tNow <= -9000.0) return;` が
  本番サーバーの待機値 `-36000.0` にも適用されていた
- `FetchShotTime()` のエラー戻り値は `-9999.0` であり、
  本番待機値 `-36000.0` は有効な値だが `-9000.0` より小さいため早期リターンされていた

**修正：**
```c
/* 修正前 */
if (tNow <= -9000.0) return;

/* 修正後 */
if (tNow == -9999.0) return;  /* FetchShotTimeエラー時のみスキップ */
```

**備考：**
- ローカルテスト（`time.txt = -5.0`）では `-5.0 > -9000.0` のため再現しなかった
- ハイブリッドテスト実施により発見・修正・動作確認済み

## v3.0.0  26.05.05

### 追加機能：ショットCSVログ

**ファイル：**
- `ThermoMonitor.c`（CSV関数追加・Timer_Callback修正・StopMonitoring修正）
- `ThermoMonitor_ui.h`（`PANEL_CSV_LOG_ENABLE` エイリアス追加）
- `ThermoMonitor_app.h`（`CsvLogEnable_Callback` プロトタイプ追加）
- `ThermoMonitor.uir`（チェックボックス追加 ← CVIエディタで手動作業）

**機能概要：**
- UIのチェックボックス「ショットCSVログ」をONにすると、SHOT毎にCSVファイルを自動生成
- 500ms毎に4CH温度データを記録
- SHOT STOP 後も記録継続（プラズマ冷却曲線を捕捉するため）
- 次の SHOT START / チェックOFF / モニタリング停止 でファイルをクローズ

**保存先：**
```
C:\Users\quest\ThermoMonitor_shot_YYYYMMDD_HHMMSS.csv
```

**CSVフォーマット：**
```csv
時刻,CH0(℃),CH1(℃),CH2(℃),CH3(℃),マーカー
11:11:00,25.2,25.3,25.2,25.0,SHOT START
11:11:00,26.1,25.3,25.2,25.0,
...
11:11:52,124.7,25.4,25.3,25.1,SHOT STOP
11:11:52,121.2,25.4,25.3,25.1,
...（冷却完了まで継続）
```

**追加した関数・変数：**

| 種別 | 名称 | 内容 |
|---|---|---|
| 静的変数 | `g_csvFile` | 現在開いているCSVファイルポインタ |
| 静的変数 | `g_csvFilePath` | 現在のCSVファイルパス |
| 静的変数 | `g_csvRowMarker` | 次のCSV行に付けるマーカー文字列 |
| 静的関数 | `OpenShotCsvFile()` | CSVファイル新規作成・ヘッダ書き込み |
| 静的関数 | `CloseShotCsvFile()` | CSVファイルクローズ・ログ出力 |
| 静的関数 | `WriteCsvRow(marker)` | 現在温度を1行書き込み |
| コールバック | `CsvLogEnable_Callback()` | チェックOFF時にCSVをクローズ |

**設計上のポイント：**
- SHOT STOPで記録を止めない理由：実験のプラズマ時間が可変・長期化傾向のため
  `time.txt` のSTOPはシーケンス制御の信号であり、温度下降タイミングと必ずしも一致しない
- `g_csvRowMarker` を介することで、SHOT START/STOP マーカーを
  `UpdateShotState()` → `Timer_Callback` の1ティック内に正確に付与できる
- UTF-8 BOM付き出力でExcelの文字化けを防止

**UIR作業：** `UIR作業手順_v300_CSVログ.md` 参照

## v3.0.2  26.05.05

### 修正：真のピーク追跡（SHOT STOP後も継続）

**ファイル：** `ThermoMonitor.c`

**症状：**
- `[PEAK TIME]` が `[SHOT STOP]` と同一時刻を示していた
- CSVログ確認で SHOT STOP後も温度が上昇し続けることが判明（145.6℃@STOP → 204.4℃@5秒後）

**原因：**
- `g_shotPeakTemp[]` の追跡が `g_shotActive==1` の期間のみ（SHOT START〜STOP）だった
- SHOT STOP信号はシーケンス制御用であり、温度ピークと必ずしも一致しない

**修正：**
- 新変数：`g_postShotTracking`・`g_truePeakTemp[]`・`g_truePeakTimeStr[][]`・`g_peakStableCount`
- SHOT START時に真のピーク追跡を開始（`g_postShotTracking=1`）
- SHOT STOP後も追跡継続（`g_postShotTracking` を維持）
- `[SHOT STOP]` はSTOP時点の現在温度（`g_lastTemp[]`）を表示
- Timer_Callback が10ティック（5秒）温度更新なしを検出したら `[PEAK TIME]` を出力して追跡終了

**ログ出力例（修正後）：**
```
[SHOT START] 12:38:00  CH0=25.2℃  ...
[SHOT STOP ] 12:38:42  CH0=145.6℃ ...  ← STOP時点の現在温度
...（追跡継続）
[PEAK TIME ] CH0=204.4℃@12:38:47  CH1=...  ← 真のピーク（CSV最高値）
```
