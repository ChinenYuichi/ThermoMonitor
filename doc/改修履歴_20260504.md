## 2. v2.8.0：CH毎Y軸スケーリング + グリッド線（2026-05-01）

**依頼内容:** CHごとにY軸の最大値・最小値を個別に設定できるようにする。

### 変更ファイル

| ファイル | 変更箇所 | 変更内容 |
|---|---|---|
| `ThermoMonitor.uir` | UIRエディタ | Numericコントロール8個追加（CH0〜CH3 × YMAX/YMIN） |
| `ThermoMonitor.c` | `ApplyChYAxisScaling()` | 新規追加 |
| `ThermoMonitor.c` | `SensorTypeSel_Callback()` | センサタイプ変更時に8コントロールへ初期値セット |
| `ThermoMonitor.c` | `Timer_Callback()` | 毎周期 `ApplyChYAxisScaling()` を呼び出す |

### 追加コントロール

| コントロールID | 役割 |
|---|---|
| `PANEL_CH0_YMAX` / `PANEL_CH0_YMIN` | CH0 Y軸最大・最小値 |
| `PANEL_CH1_YMAX` / `PANEL_CH1_YMIN` | CH1 Y軸最大・最小値 |
| `PANEL_CH2_YMAX` / `PANEL_CH2_YMIN` | CH2 Y軸最大・最小値 |
| `PANEL_CH3_YMAX` / `PANEL_CH3_YMIN` | CH3 Y軸最大・最小値 |

### 追加関数

```c
static void ApplyChYAxisScaling(void)
{
    // 各CHのYMAX/YMINコントロール値を読み取り、ストリップチャートに適用
    // yMin >= yMax の場合はスキップ（不正値ガード）
    SetAxisScalingMode(panelHandle, g_stripHandle[ch],
                       VAL_LEFT_YAXIS, VAL_MANUAL, yMin, yMax);
}
```

### 動作仕様

- センサタイプ変更時：全CHのYMAX/YMINに選択したタイプのデフォルト値をセット
- Timer_Callback 毎周期：コントロール値を読み取ってリアルタイムにグラフY軸を更新
- 各CHは独立して設定可能（他CHに影響しない）

## 3. v2.9.0：ショット連動温度ログ（2026-05-01）

**依頼内容:** 実験ショット（t=0で通電開始）の開始時刻・開始温度・ピーク温度を自動記録する。

### 参照元

- `C:\Gyrotron_Program\マニュアル・他参考資料\display_exp_status\Screen3.3r.py`
- time.txt URL: `http://csv02.exp.triam.kyushu-u.ac.jp/expinfo/time.txt`

### ショット判定ロジック

| time.txt の値 | 意味 |
|---|---|
| `t < 0` | ショット待機中 |
| `t ≥ 0` | ショット中（カウントアップ） |
| `t: 負→正` | **SHOT START** |
| `t: 正→負` | **SHOT STOP** |

### 変更ファイル

| ファイル | 変更箇所 | 変更内容 |
|---|---|---|
| `ThermoMonitor.c` | 静的変数 | `g_shotActive` / `g_prevShotTime` / `g_shotPeakTemp[]` 追加 |
| `ThermoMonitor.c` | `FetchShotTime()` | 新規追加（HTTP GET でtime.txt取得） |
| `ThermoMonitor.c` | `UpdateShotState()` | 新規追加（START/STOP検出・ログ出力） |
| `ThermoMonitor.c` | `Timer_Callback()` | `UpdateShotState()` 呼び出し追加 |

### 追加静的変数

```c
static int    g_shotActive            = 0;
static double g_prevShotTime          = -9999.0;
static double g_shotPeakTemp[SSI4C_NUM_CHANNELS];
```

### ログ出力例

```
[SHOT START] 14:32:05  CH0=23.4℃  CH1=22.8℃  CH2=22.5℃  CH3=22.6℃
[SHOT STOP ] 14:32:18  CH0=41.2℃  CH1=31.7℃  CH2=22.6℃  CH3=22.5℃  (peak)
```

### ピーク温度仕様

- ショット中に複数のピークが発生した場合、**ショット全体を通じての最高温度**を記録
- 一度記録したピーク値はより高い温度が来たときのみ上書き

## 5. v2.9.2：ショット連動ログ バグ修正2件（2026-05-01）

### バグ① ピーク追跡が機能しない

**症状:** SHOT STOP 時のピーク温度が開始温度と同じになる

**原因:** `FetchShotTime()` が -9999.0 を返すティックでは `UpdateShotState()` が早期リターンし、ピーク追跡コードがスキップされていた

**修正:** ピーク追跡を `Timer_Callback` に移動し、`FetchShotTime` の成否に依存せず毎周期実行

```c
/* Timer_Callback 内（UpdateShotState の前） */
if (g_shotActive) {
    int ch;
    for (ch = 0; ch < SSI4C_NUM_CHANNELS; ch++)
        if (g_lastTemp[ch] > g_shotPeakTemp[ch])
            g_shotPeakTemp[ch] = g_lastTemp[ch];
}
```

---

### バグ② SHOT START/STOP 検出が1〜2分遅延

**症状:** time.txt 変更後、1〜2分後に SHOT START/STOP が検出される

**原因:** Python の HTTP サーバーがヘッダーとボディを別 TCP パケットで送信することがある。1回目の `ClientTCPRead`（100ms）でヘッダーのみ受信し、ボディが空 → -9999.0 を返していた

**修正:** ボディが空の場合に2回目の `ClientTCPRead` を追加（64バイト・100ms タイムアウト）

```c
if (strlen(body) == 0) {
    char bodyBuf[64];
    memset(bodyBuf, 0, sizeof(bodyBuf));
    ClientTCPRead(handle, bodyBuf, sizeof(bodyBuf) - 1, 100);
    body = bodyBuf;
}
```

## 6. v2.9.3：DisconnectFromTCPServer -12 エラー修正（2026-05-04）

**症状:**

```
NON-FATAL RUN-TIME ERROR: "ThermoMonitor.c", line 774, col 5
Function DisconnectFromTCPServer: (return value == -12 [0xfffffff4]).
No connection established
```

**原因:** HTTP/1.0 + `Connection: close` のため、サーバーがレスポンス送信後に TCP 接続を先に閉じる。その後 `DisconnectFromTCPServer` を呼ぶと、CVI 内部では既にハンドル解放済みなので -12 が返る。

**修正方針:** TCP コールバックでサーバー切断を検知し、既に切断済みなら `DisconnectFromTCPServer` を呼ばない。

### 変更ファイル

| ファイル | 変更箇所 | 変更内容 |
|---|---|---|
| `ThermoMonitor.c` | 静的変数 | `g_fetchServerClosed` 追加 |
| `ThermoMonitor.c` | `ShotTimeTCPCB()` | 新規追加（TCP_DISCONNECT でフラグセット） |
| `ThermoMonitor.c` | `FetchShotTime()` | コールバック登録・cleanup 条件付き変更 |

### 変更詳細

```c
/* 追加：静的変数 */
static volatile int g_fetchServerClosed = 0;

/* 追加：コールバック関数 */
static int CVICALLBACK ShotTimeTCPCB(unsigned int handle, int event, int error, void *cbData)
{
    if (event == TCP_DISCONNECT)
        g_fetchServerClosed = 1;
    return 0;
}

/* 変更：FetchShotTime() 先頭 */
g_fetchServerClosed = 0;

/* 変更：ConnectToTCPServer の第4引数 */
/* 変更前 */ ConnectToTCPServer(&handle, 80, "...", NULL, NULL, 100)
/* 変更後 */ ConnectToTCPServer(&handle, 80, "...", ShotTimeTCPCB, NULL, 100)

/* 変更：cleanup: ラベル */
/* 変更前 */ DisconnectFromTCPServer(handle);
/* 変更後 */ if (!g_fetchServerClosed)
                 DisconnectFromTCPServer(handle);
```

### 動作

| 状況 | 動作 |
|---|---|
| サーバーが先に切断（通常ケース） | `TCP_DISCONNECT` → フラグ=1 → `DisconnectFromTCPServer` スキップ |
| サーバーがまだ接続中 | フラグ=0 のまま → 従来通り `DisconnectFromTCPServer` を呼ぶ |
